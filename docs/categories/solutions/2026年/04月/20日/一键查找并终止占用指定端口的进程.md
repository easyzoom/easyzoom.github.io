# 一键查找并终止占用指定端口的进程

## 前言

**C：** 本地起服务的时候，经常遇到 `Address already in use` 这种报错。多半是上一次的进程没退干净，或者另外一个工程不经意占用了同一个端口。每次都要 `lsof`、复制 PID、再 `kill`，来回三四条命令，烦得很。

于是顺手把这套动作封装成一个小函数，丢进 `~/.bashrc`，以后只要 `killport 8080` 就能收工。

<!-- more -->

## 解决方案

### 临时用法（一行命令版）

最直接的写法，适合偶尔用一下：

```sh
kill -9 $(lsof -t -i:8080)
```

- `lsof -t -i:8080`：`-t` 只输出 PID，`-i:8080` 过滤出占用 8080 端口的进程
- 外层用 `$(...)` 把 PID 作为参数传给 `kill -9`

::: warning 注意
如果端口上没有任何进程，`lsof -t` 会返回空，`kill -9` 就会提示缺少参数。所以还是建议包一层函数，先判断一下。
:::

### 常驻用法（封装成 Shell 函数）

把下面这段放到 `~/.bashrc` 或 `~/.zshrc` 的末尾：

```sh
killport() {
  if [ -z "$1" ]; then
    echo "用法: killport <端口号>"
    return 1
  fi

  local pids
  pids=$(lsof -t -i:"$1")

  if [ -z "$pids" ]; then
    echo "端口 $1 当前没有被占用。"
    return 0
  fi

  echo "端口 $1 被以下进程占用，准备结束："
  lsof -i:"$1"
  echo "$pids" | xargs kill -9
  echo "已结束 PID: $(echo "$pids" | tr '\n' ' ')"
}
```

然后 `source ~/.bashrc` 使其生效。

### 使用示例

```sh
$ killport 8080
端口 8080 被以下进程占用，准备结束：
COMMAND   PID USER   FD   TYPE             DEVICE SIZE/OFF NODE NAME
node    12345  wyh   22u  IPv6 0x...        0t0   TCP *:8080 (LISTEN)
已结束 PID: 12345

$ killport 9999
端口 9999 当前没有被占用。
```
