# U-Boot 命令系统详解

## 前言

**C：** U-Boot 的命令行虽然不像 Linux shell 那么强大，但它能做的事情比你想象的多得多——读内存、写 Flash、配网络、挂文件系统、甚至跑脚本。日常开发调试中，你大量时间都在和 U-Boot 命令打交道。本篇把常用命令分门别类讲清楚，再教你如何写自定义命令。

<!-- more -->

## 命令行基础

### 进入命令行

U-Boot 启动后，如果在 `bootdelay` 倒计时结束前按下任意键（通常是空格或回车），就会进入命令行：

```
U-Boot 2024.04 (Apr 24 2026 - 10:00:00 +0800)

DRAM:  1 GiB
Hit any key to stop autoboot:  3  2  1  0
=> _
```

### 基本操作

```bash
# 查看所有可用命令
help
# 或
?

# 查看某个命令的帮助
help bootm
help tftp

# 命令历史（上下箭头）
# Tab 补全（输入部分命令后按 Tab）
# Ctrl+C 中断正在执行的命令
# Ctrl+A 行首 / Ctrl+E 行尾
```

### 特殊语法

```bash
# 环境变量展开
echo ${bootargs}
echo ${kernel_addr_r}

# 命令分隔符（;）
setenv bootcmd 'mmc dev 1; fatload mmc 1:1 ${kernel_addr_r} Image; booti ${kernel_addr_r} - ${fdt_addr_r}'

# 命令替换：$(command)
setenv initrd_addr $(filesize)

# 条件执行
run bootcmd_mmc || run bootcmd_net

# 注释（# 开头，仅在脚本中有效）
# 这是一行注释
```

## 常用命令分类

### 信息查询类

```bash
# 版本信息
version

# 打印内存信息
bdinfo

# 查看/设置控制台波特率
baudrate

# 打印 U-Boot 编译信息
printenv
```

`bdinfo` 输出示例：

```
arch_number = 0x00000000
boot_params = 0x40000100
DRAM bank   = 0x00000000
-> start    = 0x40000000
-> size     = 0x40000000 (1 GiB)
relocaddr  = 0x7ff80000
reloc_off  = 0x3fb80000
console baudrate = 115200
model_name = Rockchip RK3399 EVB
```

### 内存操作类

这是调试中最常用的命令组：

```bash
# 内存显示（memory display）
md.b 0x40000000 0x10    # 按字节显示，从 0x40000000 开始，16 个
md.w 0x40000000 0x10    # 按半字（16-bit）
md.l 0x40000000 0x10    # 按字（32-bit）
md.q 0x40000000 0x10    # 按双字（64-bit）

# 内存写入（memory write）
mw.l 0x40000000 0x12345678 4    # 写 4 个字
mw.b 0x40000000 0xff 16         # 写 16 个字节

# 内存修改（memory modify）—— 交互式
mm.l 0x40000000         # 逐字修改

# 内存比较
cmp.l 0x40000000 0x40100000 0x100

# 内存复制
cp.b 0x40000000 0x42000000 0x1000

# 内存填充
mw.l 0x40000000 0x0 0x10000

# 循环显示（类似 watch）
md 0x40000000 10         # 只显示一次
md 0x40000000 10 1       # 每秒刷新
```

::: warning 注意

`mw` 和 `mm` 可以修改任意物理地址的内存，包括外设寄存器。**操作 MMIO 寄存器前务必确认地址正确**，写错寄存器可能导致系统挂死。

:::

### 环境变量类

```bash
# 打印所有环境变量
printenv

# 打印单个变量
printenv bootargs

# 设置变量
setenv bootargs 'console=ttymxc0,115200 root=/dev/mmcblk1p2 rootwait'

# 删除变量

# 保存到存储介质
saveenv

# 默认加载的环境变量

```

### 存储与文件系统类

```bash
# === MMC/eMMC/SD ===
mmc dev 0                   # 切换到 MMC 设备 0
mmc part                    # 显示分区表
mmc info                    # 设备信息
mmc read ${addr} ${blk} ${cnt}  # 读块
mmc write ${addr} ${blk} ${cnt} # 写块
mmc erase ${blk} ${cnt}     # 擦除
mmc bootbus 0 2 0 0         # 配置 boot bus
mmc bootpart 0 1 0          # 配置 boot partition
mmc hwpart 0 0 0            # 硬件分区切换

# === FAT 文件系统 ===
fatls mmc 0:1               # 列出 FAT 分区文件
fatload mmc 0:1 ${addr} uImage  # 加载文件到内存
fatwrite mmc 0:1 ${addr} test.bin ${filesize}  # 写文件
fatsize mmc 0:1 uImage      # 查看文件大小

# === ext4 文件系统 ===
ext4ls mmc 0:2              # 列出 ext4 分区文件
ext4load mmc 0:2 ${addr} boot/Image  # 加载文件
ext4size mmc 0:2 boot/Image  # 查看文件大小
ext4write mmc 0:2 ${addr} test.txt ${filesize}  # 写文件

# === NAND Flash ===
nand info                   # NAND 信息
nand bad                    # 显示坏块
nand read ${addr} ${off} ${size}   # 读
nand write ${addr} ${off} ${size}  # 写
nand erase ${off} ${size}          # 擦除

# === SPI Flash ===
sf probe 0                  # 探测 SPI Flash
sf read ${addr} ${off} ${len}
sf write ${addr} ${off} ${len}
sf erase ${off} ${len}
```

### 网络类

```bash
# DHCP 获取 IP
dhcp

# 手动配置网络
setenv ipaddr 192.168.1.101
setenv serverip 192.168.1.100
setenv netmask 255.255.255.0
setenv gatewayip 192.168.1.1
setenv ethaddr 00:04:9f:00:12:34

# Ping 测试
ping 192.168.1.100

# TFTP 下载
tftp ${kernel_addr_r} uImage
tftp ${fdt_addr_r} board.dtb
tftp ${initrd_addr} initramfs.cpio.gz

# NFS 挂载
nfs ${addr} 192.168.1.100:/nfs/rootfs

# 网络启动完整流程
dhcp; tftp ${kernel_addr_r} Image; tftp ${fdt_addr_r} board.dtb; booti ${kernel_addr_r} - ${fdt_addr_r}
```

### 启动类

```bash
# bootm: 启动 legacy uImage 格式
bootm ${kernel_addr_r}
bootm ${kernel_addr_r} ${ramdisk_addr_r} ${fdt_addr_r}

# bootz: 启动 ARM32 zImage
bootz ${kernel_addr_r} - ${fdt_addr_r}

# booti: 启动 ARM64 Image
booti ${kernel_addr_r} - ${fdt_addr_r}

# go: 直接跳转到内存地址执行
go 0x40800000

# bootefi: 启动 EFI 应用
bootefi ${addr}
bootefi bootmgr
```

### 设备树操作类

```bash
# 列出设备树节点
fdt addr ${fdt_addr_r}          # 设置设备树地址
fdt list /                       # 列出根节点属性
fdt list /chosen                 # 列出 chosen 节点
fdt list /soc/uart@fe650000      # 列出串口节点

# 修改设备树
fdt set /chosen bootargs 'console=ttymxc0,115200'
fdt set /soc/gpio status "okay"
fdt set /soc/mydev status "disabled"

# 删除节点
fdt rm /soc/mydev

# 创建节点
fdt mknode /soc mydevice
fdt set /soc/mydevice compatible "vendor,mydevice"

# 应用设备树覆盖（overlay）
fdt apply ${overlay_addr}

# 查看原始数据
fdt print ${fdt_addr_r} /chosen
```

## 命令别名与脚本

### run 命令

`run` 用于执行环境变量中存储的命令：

```bash
# 定义启动命令序列
setenv bootcmd_mmc 'mmc dev 0; fatload mmc 0:1 ${kernel_addr_r} Image; fatload mmc 0:1 ${fdt_addr_r} board.dtb; booti ${kernel_addr_r} - ${fdt_addr_r}'

setenv bootcmd_net 'dhcp; tftp ${kernel_addr_r} Image; tftp ${fdt_addr_r} board.dtb; booti ${kernel_addr_r} - ${fdt_addr_r}'

# 执行
run bootcmd_mmc
run bootcmd_net

# 带回退逻辑
run bootcmd_mmc || run bootcmd_net || echo "All boot methods failed"
```

### source 命令（脚本）

可以将命令序列保存为文本文件，用 `source` 执行：

```bash
# boot.scr 是编译后的 U-Boot 脚本
# 创建脚本
echo 'echo Hello; sleep 1; booti ${kernel_addr_r} - ${fdt_addr_r}' > boot.txt

# 编译为 boot.scr
mkimage -A arm64 -O linux -T script -C none -a 0 -e 0 -n "Boot Script" -d boot.txt boot.scr

# 在 U-Boot 中执行
fatload mmc 0:1 ${script_addr} boot.scr
source ${script_addr}
```

## 自定义命令开发

### 命令注册宏

```c
U_BOOT_CMD(
    name,      // 命令名
    maxargs,   // 最大参数个数
    repeatable,// 是否可重复执行（Enter 重跑）
    cmd_func,  // 命令处理函数指针
    usage,     // 简短帮助（help 输出）
    help       // 详细帮助（help <cmd> 输出）
);
```

### 实战：自定义 hello 命令

```c
// cmd/hello.c
#include <common.h>
#include <command.h>
#include <errno.h>

static int do_hello(struct cmd_tbl *cmdtp, int flag,
                     int argc, char *const argv[])
{
    if (argc == 1) {
        printf("Hello, U-Boot!\n");
    } else if (argc == 2) {
        printf("Hello, %s!\n", argv[1]);
    } else {
        return CMD_RET_USAGE;
    }
    return 0;
}

U_BOOT_CMD(
    hello,   /* 命令名 */
    2,       /* 最多 2 个参数：hello [name] */
    1,       /* 可重复 */
    do_hello,/* 处理函数 */
    "print greeting message",
    "hello [name] - print \"Hello, <name>!\" or \"Hello, U-Boot!\""
);
```

### 在 Kconfig 和 Makefile 中添加

```makefile
# cmd/Makefile（在合适的位置加一行）
obj-$(CONFIG_CMD_HELLO) += hello.o
```

```
# Kconfig（在 menuconfig 的命令菜单中加选项）
config CMD_HELLO
    bool "hello"
    help
      Print greeting message. Useful as a custom command example.
```

### 高级命令：带子命令的复合命令

U-Boot 支持子命令机制，类似于 `git` 的 `git commit`/`git push`：

```c
// cmd/mytool.c
#include <common.h>
#include <command.h>

static int do_mytool_status(struct cmd_tbl *cmdtp, int flag,
                             int argc, char *const argv[])
{
    printf("System status: OK\n");
    return 0;
}

static int do_mytool_info(struct cmd_tbl *cmdtp, int flag,
                           int argc, char *const argv[])
{
    printf("Board: %s\n", env_get("board_name"));
    printf("Arch: %s\n", env_get("arch"));
    return 0;
}

static char mytool_help_text[] =
    "status - show system status\n"
    "mytool info - show board information\n";

U_BOOT_CMD_WITH_SUBCMDS(
    mytool, "custom tool with subcommands", mytool_help_text,
    U_BOOT_SUBCMD_MKENT(status, 1, 1, do_mytool_status),
    U_BOOT_SUBCMD_MKENT(info, 1, 1, do_mytool_info)
);
```

## 命令系统内部实现

```mermaid
flowchart TD
    A["用户输入 'bootm 0x40800000'"] --> B["cli_read_line()"]
    B --> C["cli_process()"]
    C --> D["find_cmd('bootm')"]
    D --> E{命令表查找<br/>cmdtbl[]}
    E -->|找到| F["检查参数个数"]
    E -->|未找到| G["Unknown command"]
    F --> H{argc <= maxargs？}
    H -->|是| I["调用 do_bootm()"]
    H -->|否| J["CMD_RET_USAGE<br/>打印帮助"]
    I --> K["返回值处理<br/>CMD_RET_SUCCESS"]
```

U-Boot 在编译时通过 `U_BOOT_CMD` 宏把所有命令注册到一个链接器段（`.u_boot_cmd`），运行时通过哈希表或线性查找来匹配命令名。
