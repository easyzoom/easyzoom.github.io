# ROS2 安装与开发环境搭建

## 前言

**C：** ROS 2 是机器人领域的事实标准框架，但安装它往往是新手的第一道坎——选错发行版与系统组合、GPG 密钥过期、Python 版本冲突等问题层出不穷。本文从系统选择开始，逐步演示在 Ubuntu 上安装 ROS 2 的完整流程（以 Humble 和 Jazzy 为主），验证安装结果，补齐 colcon、rosdep 等开发工具，并给出 Docker 替代方案与常见问题排查指南，帮你把开发环境一次性搭稳。

<!-- more -->

## 一、系统与发行版对应关系

ROS 2 的每个发行版（Distribution）只针对特定的 Ubuntu LTS 版本做官方支持。选择系统时务必先确认对应关系，否则会遇到依赖冲突或包找不到的问题：

| ROS 2 发行版 | 对应 Ubuntu 版本 | 支持状态 | 适用场景 |
| -- | -- | -- | -- |
| Humble Hawksbill | 22.04 LTS | LTS（2027 年 5 月 EOL） | 当前生产首选，社区资源最多 |
| Iron Irwini | 22.04 | 非 LTS（已 EOL） | 过渡版本，不推荐新项目 |
| Jazzy Jalisco | 24.04 LTS | LTS（2029 年 5 月 EOL） | 新项目推荐，基于 24.04 |
| Rolling Ridley | 24.04 | 滚动更新 | 追踪最新特性，稳定性不如 LTS |

> 如果你使用其他 Linux 发行版（Debian、Fedora、RHEL 等）或 Windows/macOS，官方也提供了源码编译方式，但本文聚焦最主流的 Ubuntu + apt 方案。

::: tip 选择建议
- **工业/产品项目**：用 Humble（22.04），成熟稳定，第三方包全。
- **新启动的学习/研究项目**：用 Jazzy（24.04），生命周期更长。
- **需要最新特性**：用 Rolling，但要做好随时踩坑的准备。
:::

## 二、安装步骤

以下以 Ubuntu 22.04 + Humble 和 Ubuntu 24.04 + Jazzy 为例。如果你用其他组合，只需替换对应的发行版名称即可。

### 2.1 设置 locale

ROS 2 依赖 UTF-8 编码，先确认 locale 设置：

```bash
# 查看当前 locale
locale

# 如果没有 en_US.UTF-8，则安装并生成
sudo apt update
sudo apt install locales
sudo sed -i '/en_US.UTF-8/s/^# //g' /etc/locale.gen
sudo locale-gen en_US.UTF-8

# 设置环境变量
export LANG=en_US.UTF-8
export LANGUAGE=en_US:en
export LC_ALL=en_US.UTF-8

# 建议写入 ~/.bashrc 持久化
echo 'export LANG=en_US.UTF-8' >> ~/.bashrc
echo 'export LANGUAGE=en_US:en' >> ~/.bashrc
echo 'export LC_ALL=en_US.UTF-8' >> ~/.bashrc
source ~/.bashrc
```

### 2.2 添加 ROS 2 apt 软件源

首先安装必要的依赖并添加 ROS 2 GPG 密钥：

```bash
# 确保 apt 支持 HTTPS 和 CA 证书
sudo apt update && sudo apt install -y \
  curl gnupg lsb-release software-properties-common

# 添加 ROS 2 GPG 密钥
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg

# 添加软件源
# --- Ubuntu 22.04 + Humble ---
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
  http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" \
  | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# --- 如果是 Ubuntu 24.04 + Jazzy，命令完全相同，无需改动 ---
# 上面的命令会自动根据 $UBUNTU_CODENAME 判断系统版本
```

### 2.3 安装 ROS 2 包

```bash
# 更新软件源索引
sudo apt update

# --- Ubuntu 22.04: 安装 Humble Desktop（含 RViz、Gazebo 等 GUI 工具）---
sudo apt install -y ros-humble-desktop

# 如果你只需要命令行工具（无 GUI，适合服务器/嵌入式设备）：
# sudo apt install -y ros-humble-ros-base

# --- Ubuntu 24.04: 安装 Jazzy Desktop ---
# sudo apt install -y ros-jazzy-desktop
# sudo apt install -y ros-jazzy-ros-base
```

两个包的区别：
- `ros-<distro>-desktop`：完整桌面版，包含 RViz2、rqt、Gazebo 仿真等 GUI 工具，适合开发和学习。
- `ros-<distro>-ros-base`：最小安装，只有核心库和命令行工具，适合部署到无桌面环境的嵌入式设备。

### 2.4 设置环境变量

每次使用 ROS 2 前需要 source 环境脚本：

```bash
# Humble
source /opt/ros/humble/setup.bash

# Jazzy
# source /opt/ros/jazzy/setup.bash
```

手动每次 source 很容易忘，建议写入 `~/.bashrc`（二选一）：

```bash
# 根据你安装的发行版选择一条
echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
# echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc

# 立即生效
source ~/.bashrc
```

::: warning 多版本共存
如果你同时安装了多个 ROS 2 发行版，不要把两条 source 都写进 `.bashrc`——后加载的会覆盖前一个的环境。建议只保留你日常使用的那条，需要切换时手动 source。
:::

## 三、验证安装

ROS 2 官方提供了经典的"小海龟"demo 来验证安装是否成功。

### 3.1 运行小海龟

打开第一个终端，启动 turtlesim 节点：

```bash
# 确保已 source 环境
source /opt/ros/humble/setup.bash

# 启动小海龟仿真节点
ros2 run turtlesim turtlesim_node
```

此时应该弹出一个蓝色窗口，里面有一只小海龟。如果没弹出窗口，检查环境变量是否正确 source。

打开第二个终端，启动键盘控制节点：

```bash
source /opt/ros/humble/setup.bash

# 启动键盘控制
ros2 run turtlesim turtle_teleop_key
```

在第二个终端中按方向键，小海龟应该会移动。如果一切正常，说明 ROS 2 的节点通信、话题发布订阅都工作正常。

### 3.2 用命令行工具验证通信

在第三个终端里可以观察话题和节点信息：

```bash
source /opt/ros/humble/setup.bash

# 列出当前所有节点
ros2 node list
# 输出应包含：
#   /turtlesim
#   /teleop_turtle

# 列出当前所有话题
ros2 topic list
# 输出应包含：
#   /turtle1/cmd_vel
#   /turtle1/pose
#   ...

# 实时查看话题数据
ros2 topic echo /turtle1/pose
# 应该看到小海龟的坐标数据持续输出

# 查看节点信息图
ros2 run rqt_graph rqt_graph
# 会弹出一张可视化节点通信关系图
```

## 四、开发工具安装

仅安装 ROS 2 核心包是不够的，日常开发还需要以下工具。

### 4.1 colcon 构建工具

`colcon` 是 ROS 2 的标准构建工具，类似 ROS 1 的 `catkin_make`：

```bash
# Humble
sudo apt install -y python3-colcon-common-extensions

# Jazzy（Python 3.12 环境，包名可能有差异）
sudo apt install -y python3-colcon-common-extensions
```

安装完成后验证：

```bash
colcon version-check
```

### 4.2 rosdep

`rosdep` 用于自动安装 ROS 包的依赖，是开发中必不可少的工具：

```bash
sudo apt install -y python3-rosdep

# 初始化 rosdep
sudo rosdep init

# 更新 rosdep 数据库
rosdep update
```

之后在构建工作空间时，可以用一条命令安装所有依赖：

```bash
# 在工作空间根目录执行
rosdep install --from-paths src --ignore-src -r -y
```

### 4.3 VS Code 插件推荐

如果你使用 VS Code 作为 IDE，以下插件能显著提升 ROS 2 开发效率：

| 插件名称 | 插件 ID | 功能说明 |
| -- | -- | -- |
| ROS | ms-iot.vscode-ros | ROS/ROS2 基础支持：语法高亮、launch 配置、调试 |
| CMake Tools | ms-vscode.cmake-tools | CMake 项目管理，配合 colcon 构建 |
| Python | ms-python.python | Python 节点开发、lint、调试 |
| C/C++ | ms-vscode.cpptools | C++ 节点开发、IntelliSense |
| XML Tools | DotJoshJohnson.xml | 编辑 launch XML 文件时提供格式化和验证 |
| YAML | redhat.vscode-yaml | 编辑 YAML 参数文件时提供语法校验 |

安装完成后，在 VS Code 的设置中添加以下配置以支持 ROS 2 环境：

```json
{
  "ros.distro": "humble",
  "ros.workspace": "${workspaceFolder}"
}
```

## 五、Docker 方式使用 ROS2

如果你不想在宿主机上安装完整的 ROS 2（避免污染系统、需要多版本切换、CI/CD 场景等），Docker 是最佳选择。OSRF（Open Source Robotics Foundation）官方维护了 ROS 2 镜像。

### 5.1 拉取官方镜像

```bash
# Humble Desktop
docker pull osrf/ros:humble-desktop-full

# Humble Base（无 GUI，体积更小）
docker pull osrf/ros:humble-ros-base

# Jazzy Desktop
docker pull osrf/ros:jazzy-desktop-full
```

镜像标签说明：
- `desktop-full`：完整桌面版，含 RViz2、Gazebo、demo 等。
- `desktop`：桌面版，含 RViz2 和 rqt。
- `ros-base`：最小基础版。
- `ros-core`：仅核心通信库，体积最小。

### 5.2 运行容器

**基本运行（命令行模式）：**

```bash
docker run -it --rm osrf/ros:humble-desktop-full bash

# 进入容器后直接可以使用 ROS 2 命令
ros2 --help
```

**运行 GUI 程序（需要 X11 转发）：**

```bash
# Linux 宿主机
docker run -it --rm \
  --net=host \
  -e DISPLAY=$DISPLAY \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  osrf/ros:humble-desktop-full

# 在容器内运行小海龟
ros2 run turtlesim turtlesim_node
```

**挂载工作目录进行开发：**

```bash
# 创建工作空间目录
mkdir -p ~/ros2_ws/src

# 启动容器并挂载工作目录
docker run -it --rm \
  --net=host \
  -e DISPLAY=$DISPLAY \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  -v ~/ros2_ws:/ros2_ws \
  osrf/ros:humble-desktop-full \
  bash

# 容器内的工作空间路径就是 /ros2_ws
cd /ros2_ws
colcon build
source install/setup.bash
```

### 5.3 使用 Docker Compose 管理复杂场景

当需要多容器协作（比如 ROS 2 节点 + Gazebo 仿真 + 自定义开发容器）时，推荐使用 Docker Compose：

```yaml
# docker-compose.yml
version: "3.8"
services:
  ros2_dev:
    image: osrf/ros:humble-desktop-full
    container_name: ros2_dev
    network_mode: host
    environment:
      - DISPLAY=${DISPLAY}
    volumes:
      - /tmp/.X11-unix:/tmp/.X11-unix
      - ./ros2_ws:/ros2_ws
    stdin_open: true
    tty: true
    command: bash
```

```bash
# 启动
docker compose up -d

# 进入容器
docker compose exec ros2_dev bash
```

::: tip Docker 与真实硬件
如果需要通过 Docker 容器访问 USB 设备（如摄像头、激光雷达等），添加 `--privileged` 或使用 `--device` 参数映射具体设备：
```bash
docker run -it --rm --device=/dev/ttyUSB0 osrf/ros:humble-desktop-full bash
```
:::

## 六、常见安装问题排查

### 6.1 GPG 密钥相关错误

**现象：**

```
Err:1 http://packages.ros.org/ros2/ubuntu jammy InRelease
  The following signatures couldn't be verified because the public key is not available
```

**解决方法：**

```bash
# 重新下载并安装 GPG 密钥
sudo rm /usr/share/keyrings/ros-archive-keyring.gpg
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg

# 如果 raw.githubusercontent.com 被墙，使用镜像
sudo curl -sSL https://ghproxy.com/https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg

# 更新 apt 缓存
sudo apt update
```

### 6.2 软件源 404 错误

**现象：**

```
E: Failed to fetch http://packages.ros.org/ros2/ubuntu/dists/jammy/main/binary-amd64/Packages  404  Not Found
```

**原因：** 系统版本（`$UBUNTU_CODENAME`）与软件源中的发行版路径不匹配。比如你在 Ubuntu 24.04 上尝试安装 Humble。

**解决方法：** 明确指定系统代号，不要依赖自动检测：

```bash
# Ubuntu 24.04 安装 Jazzy，直接写 noble
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
  http://packages.ros.org/ros2/ubuntu noble main" \
  | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

sudo apt update
```

### 6.3 Python 版本冲突

**现象（Ubuntu 24.04 + Jazzy）：**

```
ModuleNotFoundError: No module named 'catkin_pkg'
```

或者 rosdep init/update 报 Python 相关错误。

**原因：** Ubuntu 24.04 默认 Python 3.12，部分 ROS 2 工具的依赖尚未完全适配。

**解决方法：**

```bash
# 确保 python3-pip 已安装
sudo apt install -y python3-pip

# 使用 pip 安装缺失的 Python 包
pip3 install --break-system-packages catkin_pkg

# 如果上述方法不行，使用 venv 隔离环境
python3 -m venv ~/ros2_venv
source ~/ros2_venv/bin/activate
pip install catkin_pkg rospkg rosdep
```

### 6.4 网络问题（中国大陆用户）

如果访问 `packages.ros.org` 或 `raw.githubusercontent.com` 速度很慢或超时：

```bash
# 方法 1：配置 apt 使用代理
sudo touch /etc/apt/apt.conf.d/proxy.conf
echo 'Acquire::http::Proxy "http://your-proxy:port";' | sudo tee /etc/apt/apt.conf.d/proxy.conf

# 方法 2：使用国内镜像源（如清华 TUNA）
# 替换软件源 URL 为：
# https://mirrors.tuna.tsinghua.edu.cn/ros2/ubuntu/

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
  https://mirrors.tuna.tsinghua.edu.cn/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" \
  | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
```

### 6.5 rosdep update 超时

**现象：**

```
ERROR: error loading sources list:
    <urlopen error [Errno 110] Connection timed out>
```

**解决方法：**

```bash
# 使用国内 rosdep 源加速
export ROSDISTRO_INDEX_URL=https://mirrors.tuna.tsinghua.edu.cn/rosdistro/index-v4.yaml

# 写入 ~/.bashrc 持久化
echo 'export ROSDISTRO_INDEX_URL=https://mirrors.tuna.tsinghua.edu.cn/rosdistro/index-v4.yaml' >> ~/.bashrc
source ~/.bashrc

# 重新初始化和更新
sudo rm -rf /etc/ros/rosdep/sources.list.d/20-default.list
sudo rosdep init
rosdep update
```

### 6.6 多个 ROS 版本环境冲突

**现象：** 命令行提示找不到某个包，或者 `ros2` 命令行为异常。

**排查方法：**

```bash
# 查看当前 source 的 ROS 2 环境来自哪里
echo $ROS_DISTRO

# 检查 AMENT_PREFIX_PATH 是否包含多个路径
echo $AMENT_PREFIX_PATH

# 如果确实存在冲突，清理环境后重新 source
unset AMENT_PREFIX_PATH
unset COLCON_PREFIX_PATH
source /opt/ros/humble/setup.bash
```

## 七、快速总结

到这里，你的 ROS 2 开发环境应该已经搭好了。回顾一下核心步骤：

1. 选择匹配的 Ubuntu + ROS 2 发行版组合
2. 设置 locale，添加软件源，安装 `ros-<distro>-desktop`
3. source 环境脚本并用小海龟 demo 验证
4. 安装 colcon、rosdep 等开发工具
5. 需要隔离环境时用 Docker 方案
6. 遇到问题时对照第六节的排查清单
