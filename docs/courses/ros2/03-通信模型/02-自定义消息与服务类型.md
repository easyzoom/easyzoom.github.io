# 自定义消息与服务类型

## 前言

**C：** ROS 2 内置了不少消息和服务类型——std_msgs 里的 String、geometry_msgs 里的 Twist、example_interfaces 里的 AddTwoInts……但对于真实项目来说，你几乎一定会遇到"内置类型不够用"的情况。比如传感器采集的数据有多个字段、自定义的服务请求需要携带复杂的参数结构。这时候就需要定义自己的 msg、srv 甚至 action 类型。本篇详细讲解如何创建独立的接口包，并在 C++ 和 Python 中使用这些自定义类型。

<!-- more -->

## 为什么需要自定义类型

内置类型覆盖了通用场景，但实际项目中经常需要：

- **结构化传感器数据**：如激光雷达点云、IMU 原始数据，字段组合是内置类型无法表达的
- **业务特定的服务**：如"查询距离最近的障碍物并返回坐标"，请求和响应都有多个字段
- **跨节点的一致性**：自定义类型可以确保多个节点使用完全相同的数据结构，减少出错

ROS 2 的最佳实践是：**将自定义接口放在独立的包中**，其他功能包通过依赖引用。这样改动接口时只需要重新编译接口包，而不影响使用方的逻辑。

## 创建自定义接口包

接口包使用 `ament_cmake` 构建，目录结构如下：

```text
my_interfaces/
├── msg/
│   └── SensorData.msg
├── srv/
│   └── GetDistance.srv
├── action/
│   └── NavigateToPoint.action
├── CMakeLists.txt
├── package.xml
```

使用 `ros2 pkg create` 创建：

```bash
ros2 pkg create --build-type ament_cmake my_interfaces
mkdir -p my_interfaces/msg my_interfaces/srv my_interfaces/action
```

## msg 文件语法

消息文件使用 `.msg` 扩展名，每行定义一个字段，格式为 `字段类型 字段名`。

```msg
# my_interfaces/msg/SensorData.msg
# 传感器数据消息

# 基本类型
uint32 timestamp_sec
uint32 timestamp_nsec

# 标准类型（引用其他 msg）
std_msgs/Header header

# 数组
float32[] distances        # 可变长度数组（最多 255 个元素）
float32[36] temperature    # 固定长度数组

# 嵌套消息（引用同包内的其他 msg）
string sensor_name
string frame_id

# 常量
uint32 MAX_READINGS = 100
```

常用字段类型：

| 类型 | 说明 |
| --- | --- |
| `bool` | 布尔值 |
| `int8/16/32/64`, `uint8/16/32/64` | 整数 |
| `float32`, `float64` | 浮点数 |
| `string` | 字符串 |
| `T[]` | 可变长度数组（T 为任意类型） |
| `T[N]` | 固定长度数组 |
| `package/MsgName` | 引用其他包的消息类型 |

## srv 文件语法

服务文件使用 `.srv` 扩展名，分为请求部分和响应部分，用 `---` 分隔：

```srv
# my_interfaces/srv/GetDistance.srv
# 查询到目标的距离

# --- 请求 ---
string target_name
string frame_id
---
# --- 响应 ---
bool success
float32 distance
string message
```

请求部分定义客户端发送的数据，响应部分定义服务端返回的数据。

## action 文件语法

动作文件使用 `.action` 扩展名，分为三段：

```action
# my_interfaces/action/NavigateToPoint.action
# 导航到指定点

# --- 目标 ---
float32 target_x
float32 target_y
string frame_id
---
# --- 结果 ---
bool success
string message
---
# --- 反馈 ---
float32 current_x
float32 current_y
float32 remaining_distance
```

## package.xml 依赖声明

```xml
<?xml version="1.0"?>
<package format="3">
  <name>my_interfaces</name>
  <version>0.1.0</version>
  <description>自定义消息、服务和动作类型</description>
  <maintainer email="you@example.com">EASYZOOM</maintainer>
  <license>Apache-2.0</license>

  <buildtool_depend>ament_cmake</buildtool_depend>

  <!-- 接口生成依赖 -->
  <buildtool_depend>rosidl_default_generators</buildtool_depend>

  <!-- 消息/服务运行时依赖 -->
  <exec_depend>rosidl_default_runtime</exec_depend>

  <!-- 接口成员的依赖 -->
  <depend>std_msgs</depend>
  <depend>geometry_msgs</depend>

  <!-- 导出 -->
  <member_of_group>rosidl_interface_packages</member_of_group>

  <export>
    <build_type>ament_cmake</build_type>
  </export>
</package>
```

关键点：
- `rosidl_default_generators`：构建时生成各语言的接口代码
- `rosidl_default_runtime`：运行时需要的库
- `member_of_group>rosidl_interface_packages`：声明这是一个接口包
- `depend`：列出了消息中引用的所有外部类型所在的包

## CMakeLists.txt 配置

```cmake
cmake_minimum_required(VERSION 3.8)
project(my_interfaces)

if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  add_compile_options(-Wall -Wextra -Wpedantic)
endif()

# 查找依赖
find_package(ament_cmake REQUIRED)
find_package(rosidl_default_generators REQUIRED)
find_package(std_msgs REQUIRED)
find_package(geometry_msgs REQUIRED)

# 生成接口
rosidl_generate_interfaces(${PROJECT_NAME}
  "msg/SensorData.msg"
  "srv/GetDistance.srv"
  "action/NavigateToPoint.action"
  DEPENDENCIES std_msgs geometry_msgs
)

ament_package()
```

::: tip 笔者说
`rosidl_generate_interfaces` 的 `DEPENDENCIES` 参数必须列出所有被引用的外部接口包，否则生成会失败。如果 msg 文件中引用了同包内的其他 msg，则不需要在 DEPENDENCIES 中声明。
:::

## 在 C++ 中使用自定义类型

首先在使用方包的 `package.xml` 中添加依赖：

```xml
<depend>my_interfaces</depend>
```

然后在 `CMakeLists.txt` 中查找依赖：

```cmake
find_package(my_interfaces REQUIRED)
ament_target_dependencies(my_node my_interfaces)
```

C++ 头文件命名规则：`包名/msg/消息名.hpp`、`包名/srv/服务名.hpp`

```cpp
#include "my_interfaces/msg/sensor_data.hpp"
#include "my_interfaces/srv/get_distance.hpp"

// 发布自定义消息
auto publisher = this->create_publisher<my_interfaces::msg::SensorData>("sensor_data", 10);
auto msg = my_interfaces::msg::SensorData();
msg.sensor_name = "lidar_front";
msg.distances = {1.2, 3.4, 5.6};
msg.header.stamp = this->now();
publisher->publish(msg);

// 发送自定义服务请求
auto client = this->create_client<my_interfaces::srv::GetDistance>("get_distance");
auto request = std::make_shared<my_interfaces::srv::GetDistance::Request>();
request->target_name = "obstacle_1";
auto future = client->async_send_request(request);
```

## 在 Python 中使用自定义类型

`package.xml` 同样需要添加依赖：

```xml
<depend>my_interfaces</depend>
```

Python 的导入方式：

```python
from my_interfaces.msg import SensorData
from my_interfaces.srv import GetDistance
from my_interfaces.action import NavigateToPoint

# 发布自定义消息
msg = SensorData()
msg.sensor_name = 'lidar_front'
msg.distances = [1.2, 3.4, 5.6]
msg.header.stamp = self.get_clock().now().to_msg()
publisher.publish(msg)

# 发送自定义服务请求
request = GetDistance.Request()
request.target_name = 'obstacle_1'
future = client.call_async(request)
```

## 编译顺序注意事项

由于功能包依赖接口包，编译时需要先编译接口包。如果你使用 `colcon build`，它会自动解析依赖并按正确顺序编译。但如果单独编译某个包，记得先编译接口包：

```bash
# 只编译接口包
colcon build --packages-select my_interfaces

# 再编译依赖它的功能包
colcon build --packages-select my_sensor_node

# 或者一次性编译全部（推荐）
colcon build
```

## 验证接口是否正确生成

编译完成后，可以用 `ros2 interface` 命令验证：

```bash
# 查看自定义消息
ros2 interface show my_interfaces/msg/SensorData

# 查看自定义服务
ros2 interface show my_interfaces/srv/GetDistance

# 查看自定义动作
ros2 interface show my_interfaces/action/NavigateToPoint
```

如果能看到完整的字段定义，说明接口包编译成功。

## 常见问题

### 修改 msg 后编译报错

修改 msg 文件后，建议执行完整清理重建：

```bash
# 删除构建产物
rm -rf build/ install/ log/

# 重新编译
colcon build
```

或者只清理接口包：

```bash
rm -rf build/my_interfaces/ install/my_interfaces/
colcon build --packages-select my_interfaces
```

### 其他包找不到自定义类型

确保：
1. `colcon build` 已成功完成（无报错）
2. 已 `source install/setup.bash`
3. 使用方包的 `package.xml` 和 `CMakeLists.txt` 中都正确声明了依赖

### 字段类型写错了怎么办

msg/srv 文件中的字段类型必须来自已安装的包。使用 `ros2 interface list` 查看所有可用的接口包和类型。

## 小结

自定义接口是 ROS 2 项目中必不可少的能力。核心要点：

1. 创建独立的接口包，使用 `ament_cmake` 构建
2. msg 定义数据结构，srv 定义请求/响应，action 定义目标/结果/反馈
3. `package.xml` 中声明 `rosidl_default_generators` 依赖和 `rosidl_interface_packages` 组
4. 使用方在 `package.xml` 和 `CMakeLists.txt`/`setup.py` 中声明对接口包的依赖
5. 编译顺序：先接口包，再功能包（`colcon build` 会自动处理）
