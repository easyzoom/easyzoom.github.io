# 多控制器协作与常用控制器配置

## 前言

**C：** 一个完整的机器人通常需要多个控制器协同工作——差速驱动控制轮子、关节状态广播器发布 TF、机械臂需要轨迹控制器。ros2_control 允许同时运行多个控制器，但也需要理解控制器之间的资源竞争和切换规则。本篇讲解常用控制器的配置和多控制器协作的方式。

<!-- more -->

## 常用控制器

| 控制器 | 类型 | 订阅 | 发布 | 用途 |
| --- | --- | --- | --- | --- |
| `diff_drive_controller` | 系统 | `/cmd_vel` | `/odom`, TF | 移动机器人 |
| `joint_state_broadcaster` | 广播器 | 无 | `/joint_states` | 发布关节状态 |
| `joint_trajectory_controller` | 轨迹 | `/joint_trajectory` | 关节命令 | 机械臂轨迹跟踪 |
| `joint_position_controller` | 位置 | `/joint_commands` | 关节命令 | 简单位置控制 |
| `forward_command_controller` | 透传 | 自定义话题 | 关节命令 | 通用命令转发 |

## joint_state_broadcaster

这是几乎所有机器人都需要的基础控制器——它读取硬件接口的状态数据，发布标准的 `/joint_states` 话题：

```yaml
joint_state_broadcaster:
  type: joint_state_broadcaster/JointStateBroadcaster
```

`robot_state_publisher` 订阅 `/joint_states` 并计算 TF。**没有它，RViz 中看不到机器人模型。**

## diff_drive_controller 详解

完整配置参数：

```yaml
diff_drive_controller:
  ros__parameters:
    left_wheel_names: ['left_wheel_joint']
    right_wheel_names: ['right_wheel_joint']

    wheel_separation: 0.45          # 轮距
    wheel_radius: 0.1               # 轮径

    publish_rate: 50.0              # odom 发布频率

    odom_topic_name: "/odom"
    odom_frame_id: "odom"
    base_frame_id: "base_footprint"

    pose_covariance_diagonal: [0.001, 0.001, 0.001, 0.001, 0.001, 0.01]
    twist_covariance_diagonal: [0.001, 0.001, 0.001, 0.001, 0.001, 0.01]

    open_loop: false                # 闭环（使用反馈）
    enable_odom_tf: true            # 发布 odom → base_link TF

    cmd_vel_timeout: 0.5            # 命令超时（秒），超时后停止
    use_stamped_vel: false          # 使用 TwistStamped 还是 Twist

    # 速度和加速度限制
    linear:
      x:
        has_velocity_limits: true
        has_acceleration_limits: true
        has_jerk_limits: false
        max_velocity: 1.0           # m/s
        min_velocity: -1.0
        max_acceleration: 2.0       # m/s²
    angular:
      z:
        has_velocity_limits: true
        has_acceleration_limits: true
        has_jerk_limits: false
        max_velocity: 2.0           # rad/s
        min_velocity: -2.0
        max_acceleration: 5.0       # rad/s²
```

## joint_trajectory_controller

用于机械臂等需要精确轨迹跟踪的场景：

```yaml
joint_trajectory_controller:
  ros__parameters:
    joints:
      - joint1
      - joint2
      - joint3

    command_interfaces:
      - position

    state_interfaces:
      - position
      - velocity

    state_publish_rate: 50.0
    action_monitor_rate: 20.0

    allow_partial_joints_goal: false
    open_loop_control: false
    constraints:
      stopped_velocity_tolerance: 0.01
      goal_time: 0.5
```

发送轨迹目标：

```bash
ros2 action send_goal /joint_trajectory_controller/follow_joint_trajectory \
  control_msgs/action/FollowJointTrajectory \
  "{trajectory: {joint_names: ['joint1', 'joint2'], points: [{positions: [1.57, 0.78], time_from_start: {sec: 2, nanosec: 0}}]}}"
```

## forward_command_controller

通用的命令转发控制器，适合简单的自定义控制：

```yaml
left_wheel_velocity_controller:
  type: forward_command_controller/ForwardCommandController
  ros__parameters:
    joints:
      - left_wheel_joint
    interface_name: velocity

right_wheel_velocity_controller:
  type: forward_command_controller/ForwardCommandController
  ros__parameters:
    joints:
      - right_wheel_joint
    interface_name: velocity
```

发布到 `/left_wheel_velocity_controller/commands` 话题即可直接控制左轮速度。

## 控制器管理

```bash
# 列出所有控制器
ros2 control list_controllers

# 列出硬件接口
ros2 control list_hardware_interfaces

# 列出可用控制器类型
ros2 control list_controller_types

# 加载并激活控制器
ros2 control load_controller --set-state active joint_state_broadcaster
ros2 control load_controller --set-state active diff_drive_controller

# 停止控制器
ros2 control set_controller_state diff_drive_controller inactive

# 卸载控制器
ros2 control unload_controller diff_drive_controller

# 列出活跃的控制器
ros2 control list_controllers --state active
```

## Launch 文件中的控制器管理

```python
from launch_ros.actions import Node
from launch import ExecuteProcess

# 使用 controller_manager 的 spawn 功能
def generate_spawn_controller_cmd(controller_name):
    return ExecuteProcess(
        cmd=['ros2', 'control', 'load_controller',
             '--set-state', 'active', controller_name],
        output='screen',
    )

return LaunchDescription([
    # 先启动 joint_state_broadcaster（几乎所有机器人都需要）
    generate_spawn_controller_cmd('joint_state_broadcaster'),
    # 再启动功能控制器
    generate_spawn_controller_cmd('diff_drive_controller'),
])
```

::: tip 笔者说
`joint_state_broadcaster` 必须在其他控制器之前启动，因为 `robot_state_publisher` 依赖它发布的 `/joint_states` 来计算 TF。TF 不正常会导致几乎所有功能都不工作。
:::

## 控制器冲突与资源管理

### 资源独占

ros2_control 中，同一个关节的同一个命令接口同时只能被一个控制器使用。例如：

```
left_wheel_joint/velocity → 被 diff_drive_controller 占用
```

如果你尝试同时启动另一个控制器也使用 `left_wheel_joint/velocity`，会报错：

```
[ERROR] [controller_manager]: 'my_controller' is not available.
  Resource 'left_wheel_joint/velocity' is already claimed by 'diff_drive_controller'
```

### 控制器切换

需要在两套控制器之间切换（如手动模式和自动导航模式）：

```bash
# 停止 A，启动 B
ros2 control set_controller_state controller_a inactive
ros2 control set_controller_state controller_b active
```

### chained controllers（链式控制器）

某些控制器需要另一个控制器的输出作为输入，形成链：

```yaml
# 例如：joint_group_velocity_controller → forward_command_controller
# velocity_controller 的输出直接写入硬件
velocity_controller:
  type: velocity_controllers/JointGroupVelocityController

# position_controller 的输出写入 velocity_controller 的输入
position_controller:
  type: position_controllers/JointGroupPositionController
  chain_joints: ['joint1', 'joint2']
  # chain_joints: 指定这些关节的命令先经过 velocity_controller
```

## 性能参数调优

```yaml
controller_manager:
  ros__parameters:
    update_rate: 50  # 控制循环频率（Hz）

    # 更高的频率 = 更平滑的控制，但更耗 CPU
    # 移动机器人：20~50 Hz 通常足够
    # 机械臂：100~1000 Hz
```

| 场景 | 推荐 update_rate |
| --- | --- |
| 移动机器人（差速） | 20 ~ 50 Hz |
| 移动机器人（全向） | 50 ~ 100 Hz |
| 机械臂（位置控制） | 100 ~ 500 Hz |
| 机械臂（力矩控制） | 500 ~ 1000 Hz |
| 四足机器人 | 200 ~ 500 Hz |

## 常见问题

### 控制器启动后没有数据

```bash
# 检查硬件接口状态
ros2 control list_hardware_interfaces

# 检查命令是否写入
ros2 topic echo /diff_drive_controller/cmd_vel_unstamped --once

# 检查 joint_states
ros2 topic echo /joint_states --once
```

### "Hardware interface already claimed"

说明同一个关节接口被多个控制器争用。解决方案：
1. 使用 `chained controllers` 模式
2. 先停止占用方控制器再启动另一个
3. 重新设计控制器分工

### 控制器参数运行时修改

```bash
# 修改参数
ros2 param set /diff_drive_controller left_wheel_names "['left_wheel_joint']"

# 从文件加载参数
ros2 param load /diff_drive_controller params.yaml
```

## 小结

多控制器协作要点：

1. **joint_state_broadcaster** 是基础，必须最先启动
2. **资源独占**：同一关节的同一接口不能被多个控制器同时使用
3. **控制器切换**：先 inactive 旧控制器，再 active 新控制器
4. **链式控制器**：控制器之间可以形成输入输出链
5. **update_rate**：根据应用场景选择合适的控制频率
6. **命令行管理**：`ros2 control load_controller` / `set_controller_state`
