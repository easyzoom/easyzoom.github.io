# ROS2 命令行工具全览

## 前言

**C：** ROS 2 的命令行工具（`ros2` 命令族）是日常开发中使用频率最高的调试手段。不用写代码、不用起 GUI，敲几个命令就能查看节点列表、话题数据、服务调用、参数设置。本篇把最常用的 `ros2` 子命令系统整理一遍，不追求面面俱到，但求覆盖日常开发 90% 的使用场景。

<!-- more -->

## ros2 命令体系概览

```bash
# 查看所有子命令
ros2 --help
```

核心子命令按功能分类：

| 分类 | 命令 | 作用 |
| --- | --- | --- |
| 节点 | `ros2 node` | 查看和管理节点 |
| 话题 | `ros2 topic` | 查看和操作话题 |
| 服务 | `ros2 service` | 查看和调用服务 |
| 动作 | `ros2 action` | 查看和触发动作 |
| 参数 | `ros2 param` | 查看和设置参数 |
| 接口 | `ros2 interface` | 查看消息/服务/动作定义 |
| 包 | `ros2 pkg` | 查看包信息 |
| Launch | `ros2 launch` | 启动 Launch 文件 |
| Daemon | `ros2 daemon` | 管理 ROS2 后台守护进程 |
| 安全 | `ros2 security` | DDS-Security 相关 |

## 节点管理（ros2 node）

```bash
# 列出所有活跃节点
ros2 node list

# 查看节点详细信息（订阅者、发布者、服务等）
ros2 node info /my_node

# 查看特定节点的订阅者
ros2 node info /my_node | grep "Subscribers:"

# 递归查看（含子节点）
ros2 node info /my_node --include-hidden
```

典型使用场景：

```bash
# 节点启动后，确认是否在运行
ros2 node list | grep my_node

# 节点没有出现在列表中？检查：
# 1. 是否 source 了 setup.bash
# 2. ROS_DOMAIN_ID 是否一致
# 3. 节点是否真的启动成功了
```

## 话题工具（ros2 topic）

话题工具是最常用的调试命令：

```bash
# 列出所有活跃话题
ros2 topic list

# 列出话题及其类型和发布者数量
ros2 topic list -v

# 查看话题类型
ros2 topic type /chatter

# 查看话题详细信息（QoS、发布者、订阅者）
ros2 topic info /chatter

# 查看 QoS 配置
ros2 topic info /chatter --verbose

# 实时打印话题数据
ros2 topic echo /chatter

# 只打印特定字段
ros2 topic echo /chatter --field data

# 打印一次就退出
ros2 topic echo /chatter --once

# 统计发布频率
ros2 topic hz /chatter

# 测量端到端延迟
ros2 topic delay /chatter

# 计算带宽占用
ros2 topic bw /chatter

# 从命令行发布消息
ros2 topic pub /chatter std_msgs/String "{data: 'hello from cli'}"

# 持续发布（指定频率）
ros2 topic pub -r 1 /chatter std_msgs/String "{data: 'hello from cli'}"

# 发布一次就退出
ros2 topic pub -1 /cmd_vel geometry_msgs/Twist "{linear: {x: 0.0}, angular: {z: 0.0}}"
```

::: tip 笔者说
`ros2 topic echo` 和 `ros2 topic pub` 是最常搭配使用的组合——一个节点 echo 监听，另一个 pub 模拟输入，快速验证通信链路。
:::

## 服务工具（ros2 service）

```bash
# 列出所有活跃服务
ros2 service list

# 列出服务及其类型
ros2 service list -t

# 查看服务类型
ros2 service type /add_two_ints

# 查找提供特定类型服务的节点
ros2 service find example_interfaces/srv/AddTwoInts

# 从命令行调用服务
ros2 service call /add_two_ints example_interfaces/srv/AddTwoInts "{a: 5, b: 3}"
```

## 动作工具（ros2 action）

```bash
# 列出所有活跃动作
ros2 action list

# 列出动作及其类型
ros2 action list -t

# 查看动作详细信息
ros2 action info /count_down

# 发送目标（不等待反馈）
ros2 action send_goal /count_down my_interfaces/action/CountDown "{target_count: 5}"

# 发送目标并查看反馈
ros2 action send_goal --feedback /count_down my_interfaces/action/CountDown "{target_count: 5}"

# 取消目标
ros2 action cancel /count_down
```

## 参数工具（ros2 param）

```bash
# 列出节点的所有参数
ros2 param list /my_node

# 获取参数值
ros2 param get /my_node control_frequency

# 设置参数值
ros2 param set /my_node debug_mode true

# 导出参数到文件
ros2 param dump /my_node

# 查看参数描述（Humble+）
ros2 param describe /my_node control_frequency
```

## 接口工具（ros2 interface）

```bash
# 列出所有接口包
ros2 interface list

# 按类型过滤
ros2 interface list msg
ros2 interface list srv
ros2 interface list action

# 查看接口定义
ros2 interface show std_msgs/msg/String
ros2 interface show example_interfaces/srv/AddTwoInts
ros2 interface show example_interfaces/action/Fibonacci

# 显示接口的包和路径
ros2 interface show std_msgs/msg/String --all-extends
```

## 包管理工具（ros2 pkg）

```bash
# 列出所有已安装的包
ros2 pkg list

# 按前缀过滤
ros2 pkg list | grep nav2

# 查看包的 prefix 路径
ros2 pkg prefix nav2_bringup

# 查看包的可执行文件
ros2 pkg executables nav2_bringup

# 创建新包
ros2 pkg create --build-type ament_cmake my_new_pkg --dependencies rclcpp std_msgs
ros2 pkg create --build-type ament_python my_new_pkg --dependencies rclpy std_msgs
```

## Daemon 管理

ROS 2 使用后台守护进程（`ros2 daemon`）来加速节点发现：

```bash
# 启动守护进程（通常自动启动）
ros2 daemon start

# 停止守护进程
ros2 daemon stop

# 重启守护进程
ros2 daemon stop && ros2 daemon start

# 查看守护进程状态
ros2 daemon status
```

::: warning 注意
如果 `ros2 node list` 等命令看不到新启动的节点，尝试重启 daemon：

```bash
ros2 daemon stop && ros2 daemon start
```

这在节点异常退出（未正确 shutdown）时尤其常见。
:::

## 高频调试技巧

### 1. 管道过滤

```bash
# 只看某个前缀的话题
ros2 topic list | grep /sensors

# 搜索包含关键字的包
ros2 pkg list | grep slam

# 查看话题频率并过滤
ros2 topic hz /chatter 2>&1 | grep "average rate"
```

### 2. 组合使用

```bash
# 终端1：监听话题
ros2 topic echo /cmd_vel

# 终端2：发送测试命令
ros2 topic pub -r 1 /cmd_vel geometry_msgs/Twist "{linear: {x: 0.5}, angular: {z: 0.0}}"

# 终端3：查看节点和话题的连接关系
ros2 topic info /cmd_vel --verbose
```

### 3. 查看通信图

```bash
# rqt_graph 可视化
ros2 run rqt_graph rqt_graph

# 命令行查看（简化版）
ros2 node list
ros2 topic list -v
```

## 常见问题

### ros2 命令报错 "command not found"

检查环境变量是否正确：

```bash
source /opt/ros/humble/setup.bash
# 或
source install/setup.bash
```

### ros2 node list 看不到节点

排查顺序：

1. 节点是否真的启动了（看终端有没有报错）
2. `source setup.bash` 了吗
3. `ROS_DOMAIN_ID` 是否一致
4. 重启 daemon：`ros2 daemon stop && ros2 daemon start`
5. 防火墙是否阻止了 DDS 通信

### 话题有数据但 echo 没有输出

可能是 QoS 不兼容。默认 `ros2 topic echo` 使用 Reliable + Volatile，如果发布者使用 BestEffort，需要：

```bash
ros2 topic echo /topic --qos-reliability best_effort
```

## 小结

ROS 2 命令行工具是日常开发最高频的调试手段。核心命令速查：

| 需求 | 命令 |
| --- | --- |
| 看节点 | `ros2 node list` / `ros2 node info` |
| 看话题 | `ros2 topic list` / `ros2 topic echo` / `ros2 topic info` |
| 发送测试数据 | `ros2 topic pub` |
| 调用服务 | `ros2 service call` |
| 查看接口定义 | `ros2 interface show` |
| 管理参数 | `ros2 param get` / `ros2 param set` / `ros2 param dump` |
