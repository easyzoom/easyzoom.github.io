# 参数系统

## 前言

**C：** 写过几个 ROS 2 节点之后你就会发现，很多值不应该硬编码在代码里——比如传感器话题名、控制频率、阈值参数。如果每次改个值都要重新编译，那开发效率就太低了。ROS 2 的参数系统提供了一套运行时可读写的配置机制，让节点可以在启动时加载配置、运行中动态修改。本篇从基本用法到高级技巧，帮你把参数用好。

<!-- more -->

## 什么是参数

参数（Parameter）是节点可以存储在服务器中的键值对。每个节点自带一个参数服务器，用于管理自己的参数。

参数的特点：

- **键值对结构**：参数名（字符串）→ 参数值（多种类型）
- **节点级别隔离**：每个节点有独立的参数空间
- **运行时可修改**：无需重新编译
- **支持回调通知**：参数变化时节点可以收到通知

支持的参数类型：

| 类型 | 说明 | 示例 |
| --- | --- | --- |
| `bool` | 布尔值 | `true` |
| `int64` | 64位整数 | `42` |
| `float64` | 64位浮点 | `3.14` |
| `string` | 字符串 | `"hello"` |
| `byte_array` | 字节数组 | `[1, 2, 3]` |
| `bool_array` | 布尔数组 | `[true, false]` |
| `int_array` | 整数数组 | `[1, 2, 3]` |
| `float_array` | 浮点数组 | `[1.0, 2.0]` |
| `string_array` | 字符串数组 | `["a", "b"]` |

## C++ 参数使用

### 声明参数

```cpp
#include <rclcpp/rclcpp.hpp>

class MyNode : public rclcpp::Node {
public:
  MyNode() : Node("my_param_node") {
    // 声明参数（含默认值）
    this->declare_parameter("control_frequency", 10.0);
    this->declare_parameter("sensor_topic", "sensor_data");
    this->declare_parameter("max_range", 5.0);
    this->declare_parameter("debug_mode", false);

    // 使用参数回调处理参数变更
    callback_handle_ = this->add_on_set_parameters_callback(
      std::bind(&MyNode::on_parameter_changed, this, std::placeholders::_1));

    // 获取参数值
    auto freq = this->get_parameter("control_frequency").as_double();
    auto topic = this->get_parameter("sensor_topic").as_string();
    RCLCPP_INFO(this->get_logger(), "控制频率: %.1f Hz", freq);
    RCLCPP_INFO(this->get_logger(), "传感器话题: %s", topic.c_str());
  }

private:
  // 参数变更回调
  rclcpp::node_interfaces::OnSetParametersCallbackHandle::SharedPtr callback_handle_;

  void on_parameter_changed(const std::vector<rclcpp::Parameter> &params) {
    for (const auto &param : params) {
      if (param.get_name() == "control_frequency") {
        double freq = param.as_double();
        if (freq <= 0.0 || freq > 1000.0) {
          RCLCPP_WARN(this->get_logger(), "无效的频率值: %.1f", freq);
          continue;
        }
        RCLCPP_INFO(this->get_logger(), "控制频率更新为: %.1f Hz", freq);
        // 这里可以更新定时器频率等
      }
    }
  }
};
```

### 通过命令行设置参数

```bash
# 方式1：启动时通过 --ros-args 设置
ros2 run my_pkg my_node --ros-args -p control_frequency:=20.0 -p debug_mode:=true

# 方式2：使用参数文件（YAML）
ros2 run my_pkg my_node --ros-params params.yaml
```

## Python 参数使用

```python
import rclpy
from rclpy.node import Node
from rcl_interfaces.msg import SetParametersResult


class MyParamNode(Node):
    def __init__(self):
        super().__init__('my_param_node')

        # 声明参数
        self.declare_parameter('control_frequency', 10.0)
        self.declare_parameter('sensor_topic', 'sensor_data')
        self.declare_parameter('max_range', 5.0)
        self.declare_parameter('debug_mode', False)

        # 参数变更回调
        self.add_on_set_parameters_callback(self.on_parameter_changed)

        # 获取参数值
        freq = self.get_parameter('control_frequency').get_parameter_value().double_value
        topic = self.get_parameter('sensor_topic').get_parameter_value().string_value
        self.get_logger().info(f'控制频率: {freq} Hz')
        self.get_logger().info(f'传感器话题: {topic}')

    def on_parameter_changed(self, params):
        for param in params:
            if param.name == 'control_frequency':
                freq = param.get_parameter_value().double_value
                if freq <= 0.0 or freq > 1000.0:
                    self.get_logger().warn(f'无效的频率值: {freq}')
                    return SetParametersResult(successful=False)
                self.get_logger().info(f'控制频率更新为: {freq} Hz')
        return SetParametersResult(successful=True)


def main(args=None):
    rclpy.init(args=args)
    node = MyParamNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

## YAML 参数文件

YAML 文件是管理多个节点参数的推荐方式：

```yaml
# params.yaml
my_param_node:
  ros__parameters:
    control_frequency: 10.0
    sensor_topic: "sensor_data"
    max_range: 5.0
    debug_mode: false

another_node:
  ros__parameters:
    target_frame: "base_link"
    timeout: 3.0
```

使用方式：

```bash
ros2 run my_pkg my_node --ros-params params.yaml
```

在 Launch 文件中加载参数文件：

```python
from launch.actions import Node
from launch.substitutions import LaunchConfiguration

Node(
    package='my_pkg',
    executable='my_node',
    parameters=['params.yaml'],
)
```

## ros2 param 命令

```bash
# 列出节点的所有参数
ros2 param list /my_param_node

# 获取参数值
ros2 param get /my_param_node control_frequency

# 设置参数值（运行时动态修改）
ros2 param set /my_param_node debug_mode true

# 导出参数到 YAML 文件
ros2 param dump /my_param_node

# 从 YAML 文件加载参数
ros2 param load params.yaml my_param_node
```

::: tip 笔者说
`ros2 param set` 修改的参数只在当前运行中生效，不会持久化。如果需要持久化，先 `ros2 param dump` 导出，下次启动时用 `--ros-params` 加载。
:::

## 参数描述（Humble 及以上版本）

从 ROS 2 Humble 开始，可以为参数添加描述信息，方便其他开发者理解参数的用途：

```cpp
// C++
#include <rcl_interfaces/msg/parameter_descriptor.hpp>

rcl_interfaces::msg::ParameterDescriptor desc;
desc.description = "传感器数据发布频率（Hz）";
desc.floating_point_range.resize(1);
desc.floating_point_range[0].from_value = 1.0;
desc.floating_point_range[0].to_value = 100.0;
this->declare_parameter("control_frequency", 10.0, desc);
```

```python
# Python
from rcl_interfaces.msg import ParameterDescriptor, FloatingPointRange

desc = ParameterDescriptor(description='传感器数据发布频率（Hz）')
desc.floating_point_range = [FloatingPointRange(from_value=1.0, to_value=100.0)]
self.declare_parameter('control_frequency', 10.0, descriptor=desc)
```

添加描述后，`ros2 param describe` 命令可以查看参数的说明和取值范围。

## 参数 vs 环境变量 vs 配置文件

| 方式 | 适用场景 | 运行时修改 |
| --- | --- | --- |
| 参数系统 | 节点级别的配置 | 支持 |
| 环境变量 | 全局配置（ROS_DOMAIN_ID 等） | 需重启 |
| YAML 配置文件 | 多节点批量配置 | 重启时生效 |
| 命令行参数 | 临时调试 | 启动时设置 |

实际项目中的推荐做法：

1. **默认值写在代码里**：`declare_parameter("freq", 10.0)`
2. **不同环境用 YAML 文件**：开发环境、测试环境、生产环境各一份
3. **临时调试用命令行**：`--ros-args -p freq:=50.0`

## 常见问题

### 参数没有声明就直接 get 会怎样

C++ 中会抛出异常，Python 中也会报错。必须先 `declare_parameter` 再 `get_parameter`。

### 参数回调中可以修改其他参数吗

在参数回调中调用 `set_parameters` 是可以的，但要注意避免递归。建议在回调中只做验证和记录，必要时在其他地方触发参数更新。

### 多个节点如何共享参数

ROS 2 没有全局参数服务器（不同于 ROS 1 的 rosparam）。每个节点的参数是独立的。如果需要共享配置，推荐使用 YAML 文件在 Launch 中统一加载。

## 小结

ROS 2 的参数系统提供了运行时可读写的配置机制，要点：

1. 参数是节点级别的键值对，支持多种类型
2. 通过 `declare_parameter` 声明并设置默认值
3. 通过 `add_on_set_parameters_callback` 监听参数变化并做验证
4. YAML 文件适合管理多节点、多环境的参数
5. 命令行 `--ros-args -p` 适合临时调试
6. 推荐实践：默认值在代码中、环境差异用 YAML、临时调试用命令行
