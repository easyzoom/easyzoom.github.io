# ROS2 包结构与创建方法

## 前言

**C：** ROS 2 的一切代码都住在"包"里——包是构建、测试、分发和重用的基本单元。不理解包的结构，后面的节点通信、launch 文件、自定义消息全都会卡在"找不到包"的错误上。本篇从包的概念出发，讲清楚 ament_cmake 和 ament_python 两种包类型的区别，用 `ros2 pkg create` 快速建包，逐字段拆解 package.xml，最后给出 C++ 和 Python 两个最小但完整的 Hello World 包示例，帮你一次性把"包"这件事彻底搞明白。

<!-- more -->

## 一、什么是 ROS2 包

在 ROS 2 生态中，**包（Package）** 是最基本的功能单元和代码组织形式。你可以把它理解为 ROS 2 世界里的"项目"或"模块"——一个包封装了一组相关的功能（节点、库、配置文件、launch 文件等），拥有独立的构建描述和依赖声明。

包承担着三个核心职责：

| 职责 | 说明 |
|------|------|
| **构建单元** | colcon 以包为单位执行编译，每个包有独立的构建配置 |
| **分发单元** | 通过 package.xml 声明元数据和依赖，可被打包、发布到 apt 仓库或二进制分发 |
| **复用单元** | 其他包通过 `<depend>` 声明来依赖你的包，构建系统自动处理链接和路径 |

一个包至少需要两个文件：`package.xml`（包的元信息）和构建配置文件（CMakeLists.txt 或 setup.py）。缺少任何一个，ament 都不认为它是一个合法的 ROS 2 包。

::: tip 核心原则
一个包 = 一个 package.xml + 一个构建配置。这是 ROS 2 最基本的约定，所有上层功能都建立在这个基础上。
:::

## 二、包类型

ROS 2 支持三种构建类型的包，对应不同的构建后端和编程语言：

### 2.1 ament_cmake（C++ 包）

- **构建后端**：ament_cmake（基于 CMake）
- **适用语言**：C++，也可以混合编译 Python 脚本
- **构建配置**：CMakeLists.txt
- **典型场景**：对性能有要求的核心算法、驱动、实时控制

这是 ROS 2 中最常见也最灵活的包类型。ament_cmake 本质上是对 CMake 的一层封装，在标准 CMake 流程的基础上增加了 ROS 2 特有的功能（如自动安装 launch 文件、导出信息等）。

### 2.2 ament_python（Python 包）

- **构建后端**：ament_python
- **适用语言**：Python 3
- **构建配置**：setup.py + setup.cfg
- **典型场景**：原型验证、数据处理节点、状态机、测试脚本

Python 包的结构遵循标准的 Python 包布局，ament_python 只是在其基础上添加了 ROS 2 的安装规则（如 resource 标记文件、console_scripts 入口点）。

### 2.3 ament_cmake 混合包

- **构建后端**：ament_cmake
- **特点**：在 CMake 包中同时编译 C++ 代码和安装 Python 模块

这种类型适合需要在同一个包中同时提供 C++ 库和 Python 接口的场景。常见于功能较复杂的基础库包，例如 TF2 相关的包。

三种类型对比：

| 特性 | ament_cmake | ament_python | ament_cmake 混合 |
|------|------------|--------------|-----------------|
| 构建系统 | CMake | setuptools | CMake |
| 主要语言 | C++ | Python | C++ + Python |
| 配置文件 | CMakeLists.txt | setup.py / setup.cfg | CMakeLists.txt |
| 灵活性 | 高 | 中 | 最高 |
| 构建速度 | 较慢（编译型） | 快（解释型） | 较慢 |

::: warning 选择建议
新手建议从纯 ament_python 包开始，快速上手后再尝试 ament_cmake。混合包只有在明确需要 C++ 和 Python 共存时才使用，不要为了"通用"而刻意选它。
:::

## 三、使用 ros2 pkg create 创建包

ROS 2 提供了 `ros2 pkg create` 命令，可以快速生成符合规范的包骨架。

### 3.1 创建 C++ 包

```bash
# 基本用法
ros2 pkg create --build-type ament_cmake <package_name>

# 完整示例（带节点）
ros2 pkg create --build-type ament_cmake \
  --node-name my_node \
  --dependencies rclcpp \
  my_cpp_pkg
```

参数说明：

| 参数 | 说明 |
|------|------|
| `--build-type` | 构建类型：`ament_cmake` 或 `ament_python` |
| `--node-name` | 自动创建一个节点源文件模板，并在 CMakeLists.txt 中添加编译目标 |
| `--dependencies` | 声明依赖包，自动写入 package.xml 和 CMakeLists.txt |
| `--library-name` | 创建一个库目标（仅 ament_cmake） |
| `--destination-directory` | 指定包的创建位置，默认为当前目录 |

生成的目录结构：

```
my_cpp_pkg/
├── CMakeLists.txt
├── package.xml
├── include/
│   └── my_cpp_pkg/
└── src/
    └── my_node.cpp
```

### 3.2 创建 Python 包

```bash
# 基本用法
ros2 pkg create --build-type ament_python <package_name>

# 完整示例（带节点）
ros2 pkg create --build-type ament_python \
  --node-name my_node \
  --dependencies rclpy \
  my_py_pkg
```

生成的目录结构：

```
my_py_pkg/
├── package.xml
├── setup.py
├── setup.cfg
├── resource/
│   └── my_py_pkg
├── test/
│   └── test_copyright.py
│   └── test_flake8.py
│   └── test_pep257.py
└── my_py_pkg/
    ├── __init__.py
    └── my_node.py
```

### 3.3 常用参数速查

```bash
# 创建一个带多个依赖的 C++ 包
ros2 pkg create --build-type ament_cmake \
  --node-name my_cpp_node \
  --dependencies rclcpp std_msgs geometry_msgs \
  my_cpp_pkg

# 在指定目录下创建 Python 包
ros2 pkg create --build-type ament_python \
  --node-name my_py_node \
  --dependencies rclpy \
  --destination-directory ~/ros2_ws/src \
  my_py_pkg

# 查看已有包的信息
ros2 pkg prefix my_cpp_pkg
ros2 pkg xml my_cpp_pkg
```

::: tip 最佳实践
使用 `--node-name` 和 `--dependencies` 参数可以一次生成可编译的包骨架，省去手动添加编译目标和依赖声明的步骤。建议养成在创建时就声明依赖的习惯。
:::

## 四、package.xml 详解

`package.xml` 是 ROS 2 包的"身份证"——它告诉构建系统和包管理器：这个包叫什么、谁维护的、依赖了哪些其他包、用什么方式构建。无论 ament_cmake 还是 ament_python，都使用相同格式的 package.xml（遵循 format 3 规范）。

### 4.1 完整字段说明

```xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <!-- 包名：必须与目录名一致 -->
  <name>my_cpp_pkg</name>

  <!-- 版本号：遵循语义化版本 MAJOR.MINOR.PATCH -->
  <version>0.1.0</version>

  <!-- 描述信息 -->
  <description>一个最小的 ROS2 C++ 示例包，用于演示包结构</description>

  <!-- 维护者信息 -->
  <maintainer email="dev@example.com">EASYZOOM</maintainer>

  <!-- 许可证 -->
  <license>Apache-2.0</license>

  <!-- 构建工具依赖 -->
  <buildtool_depend>ament_cmake</buildtool_depend>

  <!-- 构建依赖：编译时需要 -->
  <depend>rclcpp</depend>

  <!-- 仅测试依赖 -->
  <test_depend>ament_lint_auto</test_depend>
  <test_depend>ament_lint_common</test_depend>

  <!-- 导出信息 -->
  <export>
    <build_type>ament_cmake</build_type>
  </export>
</package>
```

### 4.2 依赖类型详解

`package.xml` 中有四种依赖声明，含义各不相同：

| 依赖类型 | 作用阶段 | 含义 |
|----------|---------|------|
| `<buildtool_depend>` | 构建工具 | 声明构建系统本身需要的工具，如 `ament_cmake`、`ament_python` |
| `<build_depend>` | 编译时 | 编译源码时需要头文件或链接库 |
| `<exec_depend>` | 运行时 | 运行节点时需要，但不参与编译 |
| `<test_depend>` | 测试时 | 仅在运行测试套件时需要 |

快捷标签 `<depend>` 等价于同时声明 `<build_depend>` + `<exec_depend>`，是最常用的依赖声明方式。对于大多数 ROS 2 包（rclcpp、rclpy、std_msgs 等），使用 `<depend>` 就够了。

三种典型场景的依赖声明：

```xml
<!-- 场景1：依赖一个 ROS2 包（同时需要编译和运行） -->
<depend>rclcpp</depend>

<!-- 场景2：仅在编译时需要（如代码生成工具） -->
<build_depend>rosidl_default_generators</build_depend>

<!-- 场景3：仅在运行时需要（如启动脚本依赖的包） -->
<exec_depend>tf2_ros</exec_depend>
```

### 4.3 导出信息

`<export>` 标签中最关键的是 `<build_type>`，它告诉 colcon 用什么后端来构建这个包：

```xml
<export>
  <build_type>ament_cmake</build_type>  <!-- C++ 包 -->
</export>
```

```xml
<export>
  <build_type>ament_python</build_type>  <!-- Python 包 -->
</export>
```

还可以声明其他导出信息，如文档生成插件等：

```xml
<export>
  <build_type>ament_cmake</build_type>
  <rosdoc2>rosdoc2_default</rosdoc2>
</export>
```

::: warning 常见错误
包名（`<name>`）必须与目录名完全一致，包括大小写。如果 package.xml 中写的是 `my_cpp_pkg` 但目录叫 `My_Cpp_Pkg`，colcon 会报错或无法正确安装。
:::

## 五、CMake 包完整示例

下面给出一个最小但完整的 ament_cmake 包，实现一个简单节点，每秒打印一条 Hello World 消息。

### 5.1 目录结构

```
src/my_cpp_pkg/
├── package.xml
├── CMakeLists.txt
└── src/
    └── hello_node.cpp
```

### 5.2 package.xml

```xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>my_cpp_pkg</name>
  <version>0.1.0</version>
  <description>一个最小的 ROS2 C++ 示例包</description>
  <maintainer email="dev@example.com">EASYZOOM</maintainer>
  <license>Apache-2.0</license>

  <buildtool_depend>ament_cmake</buildtool_depend>

  <depend>rclcpp</depend>

  <test_depend>ament_lint_auto</test_depend>
  <test_depend>ament_lint_common</test_depend>

  <export>
    <build_type>ament_cmake</build_type>
  </export>
</package>
```

### 5.3 CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.8)
project(my_cpp_pkg)

# 搜索 C++ 编译器，要求至少 C++14（ROS2 Humble 要求 C++17）
if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  add_compile_options(-Wall -Wextra -Wpedantic)
endif()

# 查找 ROS 2 依赖
find_package(ament_cmake REQUIRED)
find_package(rclcpp REQUIRED)

# 声明可执行目标
add_executable(hello_node src/hello_node.cpp)

# 将依赖的头文件和库链接到目标
ament_target_dependencies(hello_node rclcpp)

# 安装可执行文件到 lib/<package_name> 目录
install(TARGETS
  hello_node
  DESTINATION lib/${PROJECT_NAME}
)

# 注册 ament 包（必须放在最后）
ament_package()
```

### 5.4 src/hello_node.cpp

```cpp
#include "rclcpp/rclcpp.hpp"

int main(int argc, char *argv[]) {
    // 初始化 ROS 2
    rclcpp::init(argc, argv);

    // 创建节点，命名为 "hello_cpp_node"
    auto node = std::make_shared<rclcpp::Node>("hello_cpp_node");

    // 创建一个定时器，每秒触发一次
    auto timer = node->create_wall_timer(
        std::chrono::seconds(1),
        [&node]() {
            RCLCPP_INFO(node->get_logger(), "Hello World from C++!");
        }
    );

    // 进入事件循环，等待定时器回调
    rclcpp::spin(node);

    // 清理资源
    rclcpp::shutdown();
    return 0;
}
```

### 5.5 构建与运行

```bash
# 进入工作空间根目录
cd ~/ros2_ws

# 编译（仅编译这个包）
colcon build --packages-select my_cpp_pkg

# 加载工作空间环境
source install/setup.bash

# 运行节点
ros2 run my_cpp_pkg hello_node

# 输出（每秒一行）：
# [INFO] [hello_cpp_node]: Hello World from C++!
# [INFO] [hello_cpp_node]: Hello World from C++!
# ...
```

## 六、Python 包完整示例

下面给出一个最小但完整的 ament_python 包，实现同样的功能——每秒打印 Hello World。

### 6.1 目录结构

```
src/my_py_pkg/
├── package.xml
├── setup.py
├── setup.cfg
├── resource/
│   └── my_py_pkg
└── my_py_pkg/
    ├── __init__.py
    └── hello_node.py
```

### 6.2 package.xml

```xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>my_py_pkg</name>
  <version>0.1.0</version>
  <description>一个最小的 ROS2 Python 示例包</description>
  <maintainer email="dev@example.com">EASYZOOM</maintainer>
  <license>Apache-2.0</license>

  <buildtool_depend>ament_python</buildtool_depend>

  <depend>rclpy</depend>

  <test_depend>ament_copyright</test_depend>
  <test_depend>ament_flake8</test_depend>
  <test_depend>ament_pep257</test_depend>
  <test_depend>python3-pytest</test_depend>

  <export>
    <build_type>ament_python</build_type>
  </export>
</package>
```

### 6.3 setup.cfg

```ini
[develop]
script_dir=$base/lib/my_py_pkg
[install]
install_scripts=$base/lib/my_py_pkg
```

::: info 关键配置
这两行的作用是把生成的可执行入口脚本安装到 `lib/my_py_pkg/` 目录下。`ros2 run` 命令通过这个路径来查找可执行文件，因此 `my_py_pkg` 必须与包名一致。
:::

### 6.4 setup.py

```python
from setuptools import find_packages, setup

package_name = 'my_py_pkg'

setup(
    name=package_name,
    version='0.1.0',
    packages=find_packages(exclude=['test']),
    data_files=[
        # ament 包索引标记文件
        ('share/ament_index/resource_index/packages',
            ['resource/' + package_name]),
        # 将 package.xml 安装到 share 目录
        ('share/' + package_name, ['package.xml']),
    ],
    install_requires=['setuptools'],
    zip_safe=True,
    maintainer='EASYZOOM',
    maintainer_email='dev@example.com',
    description='一个最小的 ROS2 Python 示例包',
    license='Apache-2.0',
    tests_require=['pytest'],
    # 命令行入口：ros2 run my_py_pkg hello_node
    entry_points={
        'console_scripts': [
            'hello_node = my_py_pkg.hello_node:main',
        ],
    },
)
```

`entry_points` 中的格式为 `可执行名 = 模块路径:函数名`。`hello_node` 是通过 `ros2 run` 调用的命令名，`my_py_pkg.hello_node:main` 表示执行 `my_py_pkg/hello_node.py` 中的 `main()` 函数。

### 6.5 resource/my_py_pkg

这是一个**空文件**，但必须存在。它是 ament 的包索引标记——ament 通过扫描 `share/ament_index/resource_index/packages/` 目录来发现已安装的包。

如果忘记创建这个文件，构建虽然可能不会报错，但运行时会出现 `Package 'my_py_pkg' not found` 的错误。

```bash
# 手动创建
touch src/my_py_pkg/resource/my_py_pkg
```

### 6.6 my_py_pkg/\_\_init\_\_.py

```python
# 空文件，标记该目录为 Python 包
```

### 6.7 my_py_pkg/hello_node.py

```python
import rclpy
from rclpy.node import Node


class HelloNode(Node):
    """一个每秒打印 Hello World 的简单节点"""

    def __init__(self):
        super().__init__('hello_py_node')

        # 创建定时器，每秒触发一次回调
        self.timer = self.create_timer(1.0, self.timer_callback)
        self.get_logger().info('Hello Python Node 已启动!')

    def timer_callback(self):
        """定时器回调函数"""
        self.get_logger().info('Hello World from Python!')


def main(args=None):
    rclpy.init(args=args)
    node = HelloNode()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()


if __name__ == '__main__':
    main()
```

### 6.8 构建与运行

```bash
# 进入工作空间根目录
cd ~/ros2_ws

# 编译（使用 --symlink-install 方便开发调试）
colcon build --packages-select my_py_pkg --symlink-install

# 加载工作空间环境
source install/setup.bash

# 运行节点
ros2 run my_py_pkg hello_node

# 输出：
# [INFO] [hello_py_node]: Hello Python Node 已启动!
# [INFO] [hello_py_node]: Hello World from Python!
# [INFO] [hello_py_node]: Hello World from Python!
# ...
```

::: tip 为什么用 --symlink-install
Python 包在开发时建议始终使用 `--symlink-install` 参数。它创建符号链接而非复制文件，这样修改 Python 源码后无需重新编译，直接生效。C++ 包不存在这个选项，因为编译型语言必须重新构建。
:::

## 七、包命名规范

遵循统一的命名规范可以避免大量构建和运行时的困扰。ROS 2 官方对包名有以下约定：

### 7.1 基本规则

| 规则 | 说明 | 正确示例 | 错误示例 |
|------|------|---------|---------|
| **全小写** | 包名只允许小写字母 | `my_robot_pkg` | `MyRobotPkg` |
| **下划线分隔** | 单词之间用下划线连接 | `lidar_driver` | `lidar-driver` |
| **禁止数字开头** | 包名不能以数字开头 | `cam2_driver` | `2cam_driver` |
| **禁止空格和特殊字符** | 只允许小写字母、数字和下划线 | `usb_cam` | `usb-cam`, `usb cam` |
| **与目录名一致** | 包名必须与所在目录名完全一致 | 目录 `nav_pkg`，包名 `nav_pkg` | 目录 `nav_pkg`，包名 `Nav_Pkg` |

### 7.2 命名建议

```bash
# 好的包名：描述性强、简短、全小写下划线
my_robot_bringup      # 启动配置包
lidar_odometry        # 功能描述清晰
mobile_base_control   # 驱动/控制类

# 不好的包名：含大写、含横线、过于模糊
MyRobot-Bringup       # 大写 + 横线
utils                 # 太模糊，容易与其他包冲突
pkg1                  # 无意义命名
```

### 7.3 CMakeLists.txt 中的 PROJECT_NAME

`CMakeLists.txt` 中 `project()` 指定的名称也应与包名保持一致：

```cmake
# 正确：与目录名和 package.xml 中的 <name> 一致
project(my_cpp_pkg)

# 错误：虽然可能编译通过，但会导致安装路径混乱
project(My_Cpp_Pkg)
```

### 7.4 Python 模块名

Python 包中，`setup.py` 的 `package_name` 变量、目录名、`setup.cfg` 中的路径、`resource/` 下的标记文件名，这四处必须保持一致：

```python
# setup.py
package_name = 'my_py_pkg'        # 变量名
entry_points={
    'console_scripts': [
        'hello_node = my_py_pkg.hello_node:main',  # 模块路径
    ],
},
```

```ini
# setup.cfg
[develop]
script_dir=$base/lib/my_py_pkg    # 安装路径
```

```
# 目录结构
resource/my_py_pkg                # 标记文件
my_py_pkg/__init__.py             # Python 包目录
```

如果这四处有任何不一致，运行时就会报 `Package not found` 或 `Module not found`。这是 Python 包最容易踩的坑之一。

## 八、总结

本文覆盖了 ROS 2 包的完整知识链：

| 主题 | 关键要点 |
|------|---------|
| 包的概念 | ROS 2 的构建、分发和复用基本单元，至少需要 package.xml + 构建配置 |
| 包类型 | ament_cmake（C++）、ament_python（Python）、混合包 |
| 创建命令 | `ros2 pkg create --build-type --node-name --dependencies` |
| package.xml | 声明包名、版本、依赖（`<depend>` = build + exec）、构建类型 |
| CMake 包 | CMakeLists.txt 管理编译目标，`ament_target_dependencies` 链接依赖 |
| Python 包 | setup.py + setup.cfg + resource 标记文件，`entry_points` 定义入口 |
| 命名规范 | 全小写、下划线分隔、与目录名一致 |
