# 第一个节点：编写与运行

## 前言

**C：** 节点是 ROS 2 的灵魂——所有功能最终都运行在节点里。上一篇我们搞清楚了包的结构，这一篇直接动手写一个节点并把它跑起来。我们会分别用 C++ 和 Python 各写一个周期性打印消息的节点，走完"创建→编译→运行→调试"的完整流程，再学几个关键概念：节点命名与重映射、DDS 发现机制、以及 `ros2 node` 命令行工具的使用。掌握这些，你就拥有了 ROS 2 开发的最小闭环能力。

<!-- more -->

## 一、什么是节点

在 ROS 2 中，**节点（Node）** 是最基本的执行单元。你可以把它理解为一个独立运行的进程——每个节点有自己独立的名字、独立的内存空间，通过 ROS 2 的通信机制（话题、服务、动作）与其他节点交互。

用 Linux 的视角来看：

```
一个 ROS 2 节点 ≈ 一个 Linux 进程
```

一个机器人系统通常由多个节点组成，比如：

| 节点 | 职责 |
|------|------|
| `camera_node` | 读取摄像头图像并发布 |
| `lidar_node` | 驱动激光雷达并发布点云 |
| `slam_node` | 订阅传感器数据，执行 SLAM 建图 |
| `navigation_node` | 订阅地图和目标，规划路径 |
| `cmd_vel_node` | 订阅速度指令，驱动电机 |

这种"一个节点做一件事"的设计让系统天然具备模块化和可替换性——你可以随时替换 `camera_node` 的实现（比如从 USB 摄像头换成 MIPI 摄像头），只要它发布的话题格式不变，下游节点完全无感。

节点通过 **rcl（ROS Client Library）** 层来创建。C++ 用 `rclcpp`，Python 用 `rclpy`。它们是对底层 rcl（C 语言实现）的封装，屏蔽了 DDS 的细节。

::: tip 核心概念
节点 = 一个可执行程序 + 一个 Node 对象。Node 对象持有名字、话题、服务、定时器等所有 ROS 2 资源。`rclcpp::spin()`（或 `rclpy.spin()`）负责处理节点的事件循环，让定时器回调、话题订阅回调得以执行。
:::

## 二、C++ 节点编写

### 2.1 创建包

在上一篇文章中我们介绍了包的结构。这里直接用 `ros2 pkg create` 创建一个 C++ 包：

```bash
# 进入你的 ROS2 工作空间（假设工作空间在 ~/ros2_ws）
cd ~/ros2_ws/src

# 创建包
ros2 pkg create --build-type ament_cmake --node-name my_first_node my_first_cpp_pkg \
  --dependencies rclcpp

# 查看生成的目录结构
tree my_first_cpp_pkg
```

生成的目录结构如下：

```
my_first_cpp_pkg/
├── CMakeLists.txt
├── package.xml
├── include/my_first_cpp_pkg/
└── src/
    └── my_first_node.cpp       # 自动生成的骨架文件
```

`--dependencies rclcpp` 会自动在 `package.xml` 和 `CMakeLists.txt 中添加 rclcpp 依赖。`--node-name` 会生成一个最小的节点源文件。

### 2.2 编写节点代码

将 `src/my_first_node.cpp` 的内容替换为以下完整代码：

```cpp
#include <rclcpp/rclcpp.hpp>

// 自定义节点类，继承自 rclcpp::Node
class MyFirstNode : public rclcpp::Node {
public:
    MyFirstNode() : Node("my_first_node") {
        // 创建一个定时器，每 500ms 触发一次回调
        timer_ = this->create_wall_timer(
            std::chrono::milliseconds(500),
            std::bind(&MyFirstNode::timer_callback, this)
        );

        // 读取节点名称（演示获取节点属性）
        RCLCPP_INFO(this->get_logger(), "节点 '%s' 已启动！",
                    this->get_name());
        RCLCPP_INFO(this->get_logger(), "命名空间: '%s'",
                    this->get_namespace());
        RCLCPP_INFO(this->get_logger(), "将每 500ms 打印一条消息...");
    }

private:
    // 定时器回调函数
    void timer_callback() {
        count_++;
        RCLCPP_INFO(this->get_logger(), "Hello ROS2! 第 %ld 次问候", count_);
    }

    rclcpp::TimerBase::SharedPtr timer_;  // 定时器指针
    size_t count_ = 0;                    // 计数器
};

int main(int argc, char *argv[]) {
    // 初始化 rclcpp
    rclcpp::init(argc, argv);

    // 创建节点对象（shared_ptr，spin 时需要）
    auto node = std::make_shared<MyFirstNode>();

    // 进入事件循环，阻塞直到 rclcpp::shutdown() 被调用
    // 在这个循环中，定时器回调会被周期性执行
    rclcpp::spin(node);

    // 清理资源
    rclcpp::shutdown();
    return 0;
}
```

逐行解析关键代码：

- `class MyFirstNode : public rclcpp::Node`：继承 `rclcpp::Node` 是 ROS 2 节点的标准写法。Node 构造函数接受节点名称作为参数。
- `rclcpp::init(argc, argv)`：初始化 ROS 2 客户端库。必须在创建节点之前调用。
- `create_wall_timer()`：创建一个基于系统时钟的定时器。第一个参数是间隔时间，第二个参数是回调函数。定时器是 ROS 2 中最常用的机制之一，用于周期性执行任务（如传感器数据发布、控制循环）。
- `rclcpp::spin(node)`：进入事件循环。它本质上是调用 `epoll_wait` 或类似的机制，等待 ROS 2 事件（定时器到期、消息到达等）并分发到对应的回调函数。`spin` 会一直阻塞，直到你按 `Ctrl+C`（此时 SIGINT 信号会触发 `rclcpp::shutdown()`）。
- `rclcpp::shutdown()`：清理 ROS 2 资源。

### 2.3 配置 CMakeLists.txt

确保 `CMakeLists.txt` 中有正确的编译和安装配置：

```cmake
cmake_minimum_required(VERSION 3.8)
project(my_first_cpp_pkg)

if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  add_compile_options(-Wall -Wextra -Wpedantic)
endif()

# 查找依赖
find_package(ament_cmake REQUIRED)
find_package(rclcpp REQUIRED)

# 声明可执行目标
add_executable(my_first_node src/my_first_node.cpp)

# 链接 rclcpp 库
ament_target_dependencies(my_first_node rclcpp)

# 安装可执行文件
install(TARGETS my_first_node
  DESTINATION lib/${PROJECT_NAME}
)

ament_package()
```

关键点：`install(TARGETS ...)` 是必须的——它把编译好的可执行文件安装到 `install/lib/my_first_cpp_pkg/` 目录下，`ros2 run` 才能找到它。

## 三、Python 节点编写

### 3.1 创建包

```bash
cd ~/ros2_ws/src

ros2 pkg create --build-type ament_python --node-name my_first_node my_first_py_pkg \
  --dependencies rclpy
```

生成的目录结构：

```
my_first_py_pkg/
├── package.xml
├── setup.py
├── setup.cfg
├── resource/my_first_py_pkg          # 包标记文件
└── my_first_py_pkg/
    └── __init__.py
```

Python 包没有自动生成节点文件，需要手动创建。

### 3.2 编写节点代码

创建节点文件 `my_first_py_pkg/my_first_node.py`：

```python
#!/usr/bin/env python3
"""
我的第一个 ROS2 节点 - Python 版本
每 500ms 打印一条问候消息
"""

import rclpy
from rclpy.node import Node


class MyFirstNode(Node):
    def __init__(self):
        super().__init__('my_first_node')

        self.count = 0

        # 创建定时器，每 0.5 秒触发一次
        self.timer = self.create_timer(0.5, self.timer_callback)

        self.get_logger().info(f"节点 '{self.get_name()}' 已启动！")
        self.get_logger().info(f"命名空间: '{self.get_namespace()}'")
        self.get_logger().info('将每 500ms 打印一条消息...')

    def timer_callback(self):
        self.count += 1
        self.get_logger().info(f'Hello ROS2! 第 {self.count} 次问候')


def main(args=None):
    # 初始化 rclpy
    rclpy.init(args=args)

    # 创建节点
    node = MyFirstNode()

    # 进入事件循环，处理定时器和回调
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass  # Ctrl+C 退出时 rclpy 已经处理了

    # 销毁节点并清理
    node.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()
```

Python 版本和 C++ 版本的逻辑完全一致，关键区别：

| 对比项 | C++ (rclcpp) | Python (rclpy) |
|--------|-------------|----------------|
| 初始化 | `rclcpp::init(argc, argv)` | `rclpy.init(args=args)` |
| 创建节点 | `std::make_shared<Node>(...)` | `Node(...)` |
| 定时器 | `create_wall_timer(chrono, callback)` | `create_timer(seconds, callback)` |
| 事件循环 | `rclcpp::spin(node)` | `rclpy.spin(node)` |
| 日志 | `RCLCPP_INFO(logger, fmt, ...)` | `self.get_logger().info(msg)` |
| 清理 | 自动 RAII + `rclcpp::shutdown()` | `node.destroy_node()` + `rclpy.shutdown()` |

### 3.3 配置 setup.py

确保 `setup.py` 中正确声明了入口点：

```python
from setuptools import setup, find_packages
import os
from glob import glob

package_name = 'my_first_py_pkg'

setup(
    name=package_name,
    version='0.1.0',
    packages=find_packages(exclude=['test']),
    data_files=[
        ('share/ament_index/resource_index/packages',
            ['resource/' + package_name]),
        ('share/' + package_name, ['package.xml']),
    ],
    install_requires=['setuptools'],
    zip_safe=True,
    maintainer='your_name',
    maintainer_email='your_email@example.com',
    description='我的第一个 ROS2 Python 节点',
    license='Apache-2.0',
    entry_points={
        'console_scripts': [
            'my_first_node = my_first_py_pkg.my_first_node:main',
        ],
    },
)
```

`console_scripts` 中的 `my_first_node` 是你用 `ros2 run` 时指定的可执行名称，等号右侧是 `模块路径:main函数`。

给节点文件加上可执行权限：

```bash
chmod +x my_first_py_pkg/my_first_node.py
```

## 四、编译与运行

### 4.1 编译

```bash
cd ~/ros2_ws

# 编译所有包（也可以指定包名：--packages-select my_first_cpp_pkg）
colcon build

# 如果只编译新包，用 --symlink-install 加速 Python 开发
colcon build --symlink-install
```

编译成功后会看到：

```
Starting >>> my_first_cpp_pkg
Starting >>> my_first_py_pkg
Finished <<< my_first_cpp_pkg [3.52s]
Finished <<< my_first_py_pkg [0.61s]
Summary: 2 packages finished [4.28s]
```

::: warning 重要：source install/setup.bash
每次编译后（或者打开新终端时），都必须 source 工作空间的 setup.bash，否则 `ros2 run` 找不到你编译的包：

```bash
source ~/ros2_ws/install/setup.bash
```

这是初学者最容易踩的坑。如果你遇到 `package 'my_first_cpp_pkg' not found`，90% 的概率是忘记 source 了。
:::

### 4.2 运行节点

```bash
# 运行 C++ 节点
ros2 run my_first_cpp_pkg my_first_node

# 在另一个终端运行 Python 节点
ros2 run my_first_py_pkg my_first_node
```

`ros2 run` 的命令格式为：

```
ros2 run <包名> <可执行名称>
```

运行后你会看到类似输出：


按 `Ctrl+C` 停止节点。

## 五、使用 ros2 node 命令查看节点

ROS 2 提供了一组命令行工具来 inspect 运行中的系统。`ros2 node` 是其中最常用的之一。

### 5.1 列出所有运行中的节点

```bash
# 先在终端1启动节点
ros2 run my_first_cpp_pkg my_first_node

# 在终端2执行
ros2 node list
```

输出：

```
/my_first_node
```

如果同时运行了 C++ 和 Python 版本，它们默认同名，你会只看到一个。解决方法参见下一节"节点命名与重映射"。

### 5.2 查看节点详细信息

```bash
ros2 node info /my_first_node
```

输出：

```
/my_first_node
  Subscribers:
    /parameter_events: rcl_interfaces/msg/ParameterEvent
  Publishers:
    /parameter_events: rcl_interfaces/msg/ParameterEvent
    /rosout: rcl_interfaces/msg/Log
  Service Servers:
    /my_first_node/describe_parameters: rcl_interfaces/srv/DescribeParameters
    /my_first_node/get_parameter_types: rcl_interfaces/srv/GetParameterTypes
    /my_first_node/get_parameters: rcl_interfaces/srv/GetParameters
    /my_first_node/list_parameters: rcl_interfaces/srv/ListParameters
    /my_first_node/set_parameters: rcl_interfaces/srv/SetParameters
    /my_first_node/set_parameters_atomically: rcl_interfaces/srv/SetParametersAtomically
  Service Clients:
  Action Servers:
  Action Clients:
```

即使我们只创建了一个定时器，节点自动注册了一系列参数相关的服务（每个 ROS 2 节点都自带参数服务）。这里展示了：

- **Subscribers**：该节点订阅的话题
- **Publishers**：该节点发布的话题（`/rosout` 用于日志）
- **Service Servers**：该节点提供的服务
- **Action Servers/Clients**：动作相关（后续章节详述）

## 六、节点命名与重映射

### 6.1 节点名称规则

ROS 2 的节点名称有以下规则：

| 规则 | 说明 | 示例 |
|------|------|------|
| 必须以 `/`、`~` 或字母开头 | 不能以数字或特殊字符开头 | `/camera`、`~lidar`、`motor` |
| 推荐使用下划线分隔 | 不用空格和连字符 | `my_node`（不是 `my-node`） |
| 完全限定名（FQN） | 带命名空间前缀 | `/ns1/my_node` |
| 相对名 | 不以 `/` 开头，自动拼接到命名空间 | `my_node` → `/<namespace>/my_node` |
| 私有名（不推荐在 ROS 2） | 以 `~` 开头 | `~my_node`（ROS 2 中行为有变） |

### 6.2 通过命令行重映射节点名

在实际开发中，你可能需要同时运行同一个节点的多个实例（比如多个摄像头）。ROS 2 允许在运行时重映射（remap）节点名称：

```bash
# 方法1：使用 --ros-args
ros2 run my_first_cpp_pkg my_first_node --ros-args -r __node:=camera_front

# 方法2：重命名后查看
ros2 node list
```

输出：

```
/camera_front
```

常用重映射参数：

```bash
# 重映射节点名
ros2 run <pkg> <node> --ros-args -r __node:=new_name

# 设置命名空间
ros2 run <pkg> <node> --ros-args -r __ns:=/sensors

# 同时设置命名空间和节点名
ros2 run <pkg> <node> --ros-args -r __ns:=/sensors -r __node:=camera_front
# 节点全名：/sensors/camera_front

# 通过命令行参数设置日志级别
ros2 run <pkg> <node> --ros-args --log-level debug
```

::: tip 实践建议
在嵌入式系统中，经常需要在一块板子上跑多个同类传感器节点。善用 `--ros-args -r __ns:=` 和 `-r __node:=` 可以避免命名冲突，是 ROS 2 多节点部署的基础能力。
:::

## 七、节点间的发现机制

### 7.1 DDS Discovery

这是 ROS 2 和 ROS 1 最根本的架构差异之一。ROS 1 中所有节点必须连接到 `rosmaster`，由 master 协调注册和发现。ROS 2 基于 DDS（Data Distribution Service），采用**分布式自动发现**：

```
ROS 1:  节点A → rosmaster ← 节点B    （中心化，单点故障）
ROS 2:  节点A ←→ DDS 网络 ←→ 节点B   （去中心化，自动发现）
```

当你在终端执行 `ros2 run` 启动一个节点时，底层发生了这些事：

1. `rclcpp::init()` 初始化 DDS 域参与者（DomainParticipant）
2. 节点通过 DDS 的内置主题（Built-in Topics）广播自己的存在
3. 其他节点监听到广播后，自动发现并建立通信连接

这个过程对开发者完全透明——你不需要手动注册、不需要知道对方的 IP 地址（同一网络内即可自动发现）。

### 7.2 ROS_DOMAIN_ID 的作用

DDS 使用 **Domain ID** 来隔离不同的通信域。ROS 2 暴露了这个机制：

```bash
# 查看当前 Domain ID（默认是 0）
echo $ROS_DOMAIN_ID

# 设置 Domain ID 为 1
export ROS_DOMAIN_ID=1
ros2 run my_first_cpp_pkg my_first_node
```

Domain ID 的作用：

| 场景 | 说明 |
|------|------|
| 默认值 0 | 所有未设置 ROS_DOMAIN_ID 的节点互通 |
| 不同 ID 隔离 | ID=1 和 ID=2 的节点完全看不到对方 |
| 同一网络多机器人 | 每台机器人使用不同的 Domain ID，避免串台 |

::: warning 注意
Domain ID 范围是 0-232。同一台机器上的所有 ROS 2 节点必须使用相同的 Domain ID 才能互相发现。如果你 `ros2 node list` 看不到其他节点，首先检查双方的 `ROS_DOMAIN_ID` 是否一致。
:::

此外，ROS 2 还提供了更精细的网络隔离方式——通过 `ROS_AUTOMATIC_DISCOVERY_RANGE` 可以限制节点只在特定子网或本地回环发现。对于嵌入式设备调试，这个功能很实用：

```bash
# 只在本机发现（不跨网络广播，适合调试）
export ROS_AUTOMATIC_DISCOVERY_RANGE=LOCALHOST

# 只在特定子网发现
export ROS_AUTOMATIC_DISCOVERY_RANGE=SUBNET
```

## 八、常见问题

### 8.1 编译失败：找不到 rclcpp

```
Could not find a package configuration file provided by "rclcpp"
```

**原因**：ROS 2 没有正确安装，或者 `AMENT_PREFIX_PATH` 没有设置。

**解决**：

```bash
source /opt/ros/humble/setup.bash
colcon build
```

### 8.2 运行时找不到包

```
Package 'my_first_cpp_pkg' not found
```

**原因**：忘记 source 工作空间的 `install/setup.bash`。

**解决**：

```bash
source ~/ros2_ws/install/setup.bash
ros2 run my_first_cpp_pkg my_first_node
```

### 8.3 ros2 node list 看不到节点

**可能原因及排查步骤**：

1. **Domain ID 不一致**：

   ```bash
   # 在两个终端分别检查
   echo $ROS_DOMAIN_ID
   ```

2. **忘记 source**：新终端没有 source install/setup.bash。

3. **节点还没完全启动**：节点从启动到被发现通常需要 1-2 秒，稍等再试。

4. **DDS 实现冲突**：不同终端 source 了不同 ROS 2 发行版（比如 Humble 和 Iron 混用），导致 DDS 实现不一致。

5. **防火墙**：Linux 防火墙可能阻止 DDS 的多播通信：

   ```bash
   # 临时关闭防火墙（测试用，不要在生产环境这样做）
   sudo ufw disable
   ```

### 8.4 Python 节点报 ModuleNotFoundError

```
ModuleNotFoundError: No module named 'my_first_py_pkg'
```

**原因**：Python 包没有正确安装。确保 `setup.py` 中的 `packages=find_packages()` 能找到你的模块，并且编译时使用了 `--symlink-install`（开发阶段推荐）。

```bash
colcon build --packages-select my_first_py_pkg --symlink-install
source install/setup.bash
```

### 8.5 两个同名节点只能看到一个

这是正常行为。ROS 2 中节点名称必须唯一，后启动的同名节点会导致前一个被踢出发现列表（具体行为取决于 DDS 实现）。解决方法是在启动时用 `--ros-args -r __node:=` 给每个实例不同的名字。

### 8.6 Ctrl+C 无法退出节点

偶尔在嵌入式环境中会遇到。可以尝试：

```bash
# 发送 SIGINT（等同于 Ctrl+C）
kill -SIGINT <pid>

# 如果还不行，强制终止
kill -9 <pid>
```
